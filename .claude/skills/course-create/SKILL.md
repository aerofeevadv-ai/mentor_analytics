# Course Create — Оркестратор создания курса 20/80

Полный pipeline создания курса: исследование -> структура -> контент -> проверка качества. Сохраняет 3-шаговый workflow (structure -> sections -> guide) из courses/CLAUDE.md, усиливая каждый шаг.

## Контекст

- **Формат:** 20% теории -> 80% действий и примеров
- **Аудитория:** ученики менторства + внешние покупатели
- **Тон:** практичный, конкретный, без воды
- **Workflow курсов:** `courses/CLAUDE.md` (ОБЯЗАТЕЛЬНО прочитать перед началом)

---

## Режимы запуска

### Режим A: Новый курс с нуля
Пользователь: «делаем курс по [теме]» -> начать с Шага 1

### Режим B: Написать раздел существующего курса
Пользователь: «пишем раздел [N] курса [тема]» -> перейти к Шагу 4

### Режим C: Финализировать курс
Пользователь: «собираем гайд курса [тема]» -> перейти к Шагу 5

---

## Workflow: 7 шагов

```
1. ИССЛЕДОВАНИЕ    -> что уже есть по теме, конкуренты, лучшие практики
2. ЭКСПЕРТНЫЙ ВВОД -> спросить Алексея про опыт и позицию по теме
   | СТОП -- ждём ответ
3. СТРУКТУРА       -> structure.md (на согласование)
   | СТОП -- ждём ОК
4. РАЗДЕЛЫ         -> sections/XX_название.md (на согласование)
   | СТОП -- ждём ОК
5. НАПИСАНИЕ       -> полный текст каждого раздела
6. ПРОВЕРКА        -> factcheck + качество + полнота
7. СБОРКА          -> guide.md + scripts/
```

---

## Шаг 1: Исследование темы

Перед созданием структуры -- понять ландшафт:

1. **Что уже есть в репо:** проверить `courses/` на существующие материалы по теме
2. **Конкуренты:** поискать топ-3 курса/гайда по теме (Karpov.Courses, Stepik, YouTube)
3. **Собесы:** какие вопросы реально спрашивают на собесах по этой теме (из опыта менторства: 47 учеников, 18 офферов)
4. **Пробелы:** что конкуренты НЕ покрывают (наша возможность)

Вывод: краткий отчёт (5-7 пунктов) -- что включать, что пропустить, чем отличаться.

---

## Шаг 2: Экспертный ввод (СТОП -- спросить Алексея)

```
Тема курса: [тема]

Я исследовал тему. Вот что конкуренты покрывают: [список].
Вот что НЕ покрывают: [список].

Вопросы:
1. Какие темы из этого списка реально спрашивают на собесах?
2. Что из своего опыта (Яндекс, WB) хочешь включить как кейсы?
3. Какой уровень глубины? (для джуна / для мидла / оба)
4. Есть ли «горячие» вопросы, которые ученики задают чаще всего?
5. Какую позицию занимаешь по спорным моментам в теме?
```

**СТОП.** Дождаться ответа. Использовать ответы для формирования структуры.

---

## Шаг 3: Структура (на согласование)

Создать `courses/[тема]/structure.md` по формату из courses/CLAUDE.md.

Дополнительно к стандартному формату:

```markdown
## Конкурентное отличие
[Чем наш курс отличается от существующих. 2-3 пункта]

## Типичные вопросы с собесов (покрытие)
- [Вопрос 1] -> Раздел N
- [Вопрос 2] -> Раздел M
```

**СТОП.** Дождаться согласования.

---

## Шаг 4: Разделы (для каждого)

Для каждого раздела -- создать `courses/[тема]/sections/NN_название.md` по формату из courses/CLAUDE.md.

**Усиление каждого раздела:**

1. **Answer-first:** каждый раздел начинается с ответа, потом объяснение
2. **Кейс из практики:** минимум 1 реальный пример на раздел (из опыта менторства, Яндекс, WB)
3. **Вопрос с собеса:** если раздел покрывает тему собесов -- добавить типовой вопрос + разбор
4. **Практика:** конкретные задания, не «попробуйте сами»

**СТОП.** Показать все разделы на согласование.

---

## Шаг 5: Написание полного текста

После согласования всех sections -- писать guide.md.

**Правила:**
- Читать каждый section перед написанием его части guide
- Формат: Notion-совместимый Markdown (из courses/CLAUDE.md)
- Каждый H2-раздел: тезис -> объяснение -> пример -> практика
- Код в code blocks с подсветкой синтаксиса
- Таблицы для сравнений и чек-листов
- Callout-блоки: важное, типичная ошибка, пример, практика

**Для кода (SQL, Python):**
- Каждый пример -- рабочий, с реальными данными
- Комментарии в коде -- минимум, только неочевидное
- Сначала простой вариант, потом продвинутый (если есть)

---

## Шаг 6: Проверка качества

После написания guide.md -- провести проверку по 5 категориям:

| Категория | Вес | Что проверяем |
|---|---|---|
| **Полнота** | 25% | Все разделы из structure.md покрыты? Нет пропусков? |
| **Практичность** | 25% | Каждый раздел имеет пример + задание? Можно применить сразу? |
| **Собесная готовность** | 20% | Покрыты типовые вопросы? Ученик готов ответить? |
| **Тон и стиль** | 15% | Без воды? Конкретно? Как объясняешь другу? |
| **Техническая корректность** | 15% | Код рабочий? Формулы правильные? Нет устаревшего? |

**Scoring:**
- 85-100: публиковать
- 70-84: доработать конкретные блоки
- <70: переписать слабые разделы

Для каждой проблемы -- конкретная правка с «БЫЛО -> СТАЛО».

---

## Шаг 7: Сборка финального пакета

1. Финализировать `guide.md`
2. Определить где нужно видео (по правилам из courses/CLAUDE.md)
3. Написать `scripts/intro_script.md` (онбординг, всегда)
4. Написать `scripts/screencast_NN.md` для разделов, где нужен скринкаст
5. Показать итоговую структуру:

```
courses/[тема]/
  structure.md          -- СОГЛАСОВАНО
  sections/             -- СОГЛАСОВАНО
  guide.md              -- ГОТОВО (XX страниц)
  scripts/
    intro_script.md     -- ГОТОВО (~X мин)
    screencast_01.md    -- ГОТОВО (~X мин)
  
Quality score: XX/100
Разделов: N
Кейсов из практики: N
Вопросов с собесов: N
```
