# TG Batch — Пакетная генерация постов на 2 недели

Создаёт 6-8 постов для Telegram-канала за один сеанс. Каждый пост проходит полный пайплайн: написание, AI-проверка, качество-проверка.

## Контекст

- **Канал:** Аналитика без воды: деньги и офферы
- **Автор:** Алексей Ерофеев, ментор аналитиков
- **Промт копирайтера:** `content/telegram/claude_prompt.md` (ОБЯЗАТЕЛЬНО прочитать перед генерацией)
- **Расписание:** `content/CLAUDE.md`

---

## Расписание постов

| День | Нечётная неделя | Чётная неделя |
|---|---|---|
| **Вт** | Кейс ученика | Разбор задачи с собеса |
| **Ср** | Полезный (совет, чек-лист, ошибка) | Полезный (мини-разбор, мнение) |
| **Вс** | Личный инсайт | Лёгкий / ситуативный |

Четверг (анонс YouTube) НЕ входит в batch -- его публикуют по факту выхода видео.

---

## Workflow

### ШАГ 0: Подготовка

1. Прочитать `content/telegram/claude_prompt.md` — правила тона и AI-маркеров
2. Прочитать `content/telegram/ideas.md` — есть ли готовые идеи
3. Определить текущую дату и неделю (чётная/нечётная)
4. Посмотреть последние посты в `content/telegram/posts/` — не повторять темы
5. Составить план на 2 недели (6 постов):

```
Неделя N (нечётная/чётная):
  Вт [дата]: [тип] — [тема]
  Ср [дата]: [тип] — [тема]
  Вс [дата]: [тип] — [тема]

Неделя N+1 (чётная/нечётная):
  Вт [дата]: [тип] — [тема]
  Ср [дата]: [тип] — [тема]
  Вс [дата]: [тип] — [тема]
```

### ШАГ 1: Спросить Алексея о фактуре

**СТОП.** Показать план и спросить:

```
Вот план постов на 2 недели:
[план]

Вопросы:
1. Есть ли свежие кейсы учеников для вторника? (имя не будет упоминаться)
2. Есть ли задача с недавнего собеса для разбора?
3. Какие-то темы хочешь заменить или добавить?
4. Есть ли личный инсайт/наблюдение для воскресенья?
```

Дождаться ответа. Если Алексей говорит «сам придумай» — использовать ideas.md и типовые кейсы (47 учеников, 18 офферов, средний оффер 226к).

### ШАГ 2: Генерация постов

Для каждого поста из плана:

1. **Написать пост** по правилам из `content/telegram/claude_prompt.md`
2. **Проверить на AI-маркеры** (категории A, B, C из tg-post-create):
   - Категория A (структурные): нет «Вывод:», нет эмодзи-заголовков, нет шаблонных footer
   - Категория B (языковые): нет «Лайфхак:», нет длинных тире, нет AI-слов
   - Категория C (формат): есть голос автора, неровные абзацы, есть шероховатости
   - AI-score должен быть <= 3
3. **Проверить качество** глазами джуна:
   - Хук: зацепило?
   - Понятность: термины ясны?
   - Полезность: узнал новое?
   - Характер: чувствуется автор?
   - Quality-score должен быть >= 8
4. Если не проходит проверки — переписать (макс 3 итерации)

### ШАГ 3: Сохранение

Каждый пост сохранить в файл:

```
content/telegram/posts/YYYY-MM-DD_slug.md
```

Формат файла:

```markdown
# Пост: [Тема]

**Тип:** кейс / разбор задачи / полезный / инсайт / лёгкий
**Дата публикации:** YYYY-MM-DD
**Площадки:** Telegram, MAX, VK группа

---

[ТЕКСТ ПОСТА]

---

## Проверки
AI-score: X/10
Quality-score: X/10
Длина: X символов

## Статус
ГОТОВО
```

## Правила разнообразия

- **Не начинать** два поста подряд одинаково (с вопроса, с «Вчера...», с факта)
- **Варьировать длину:** минимум 1 короткий (<400 симв.) и 1 длинный (>1000 симв.) в batch
- **Варьировать формат кейсов:** длинная история / короткий факт / от лица ученика / разбор «что делали»
- **Не повторять CTA:** подписка / реакция / вопрос в комменты / ничего — чередовать
- **Проверить по идеям:** если тема есть в `content/telegram/ideas.md` — вычеркнуть после использования
