# Skill: Проверка на AI-маркеры в TG-посте

## Описание

Проверить пост на признаки AI-генерации. Цель — чтобы пост читался как написанный человеком, а не ChatGPT/Claude.

## Вызов

```
/tg-ai-check
```

Затем вставить текст поста.

## Чек-лист AI-маркеров

### Категория A: Структурные маркеры (критичные)

| # | Маркер | Пример | Статус |
|---|---|---|---|
| A1 | «💬 Главный вывод:» / «💬 Вывод:» | Был в 35 из 60 постов канала | |
| A2 | Эмодзи в начале заголовка | 🔥 Как выучить SQL за месяц | |
| A3 | 🎯 как секционный маркер | 🎯 Результат — оффер 250к | |
| A4 | 1️⃣ 2️⃣ 3️⃣ вместо обычных цифр | 1️⃣ Шаблоны запросов | |
| A5 | ❌/✅ как буллеты | ❌ Scikit-learn ✅ Pandas | |
| A6 | Шаблонный footer кейса | 🟦 Источник... 🟣 Серия... | |

### Категория B: Языковые маркеры (заметные)

| # | Маркер | Пример |
|---|---|---|
| B1 | «Лайфхак:» | Лайфхак: на собесе всегда уточняйте... |
| B2 | «Что важно:» / «Что это значит:» | Что это значит: HR в Т-банке... |
| B3 | «Помните:» + мотивация | Помните: ваша зарплата = навыки × продажа |
| B4 | «Давайте разберёмся» | Давайте разберемся без воды |
| B5 | «Простыми словами:» | Простыми словами: p-value — это... |
| B6 | Мотивационное закрытие | Сезон открыт. Пора действовать. |
| B7 | «Без воды» в тексте | ...разбираю без воды, чтобы... |
| B8 | «Именно за это и платят X00к» | ...именно за это и готовы платить 300к+ |

### Категория C: Паттерны формата (субтильные)

| # | Маркер | Описание |
|---|---|---|
| C1 | Одинаковая длина | Каждый пост 800-1500 сим, нет вариации |
| C2 | Идеальная структура | Тезис → список → вывод, без отступлений |
| C3 | Нет разговорной «грязи» | Нет скобок, оговорок, сленга |
| C4 | Перечисление с объяснением | «X — это Y. Z — это W.» последовательно |
| C5 | Безличность | Мог написать кто угодно, нет голоса автора |

## Формат ответа

```
## Проверка на AI-маркеры

**Найдено маркеров:** X

### Критичные (категория A):
- [A1] ✅ чисто / ❌ найдено — [цитата]
- [A2] ✅ / ❌ — [цитата]
...

### Языковые (категория B):
- [B1-B8] ✅ / ❌ — [цитата если найдено]

### Формат (категория C):
- [C1-C5] ✅ / ❌ — [комментарий]

**AI-score:** X/10 (0 = полностью человеческий, 10 = очевидно AI)

**Что исправить:**
1. [конкретная правка с примером до/после]
```

## Примеры

### Плохо (AI-score 8/10):
```
🧹 80% работы аналитика — подготовка данных

🎯 Типичные грязные паттерны:
— Дубли из-за повторной отправки событий
— NULL и пустые строки вперемешку

🎯 Как ускориться:
1️⃣ Шаблоны запросов.
2️⃣ Проверки на входе.

💬 Главный вывод:
Чистые данные — фундамент любого анализа.
```

### Хорошо (AI-score 2/10):
```
Вчера полдня искал, почему отчёт показывает х2 по выручке.

Оказалось — дубли. Событие отправлялось дважды после
последнего релиза. Классика.

У меня есть шаблонный CTE, который я вставляю в каждый
запрос перед анализом — дедупликация по event_id + user_id
+ timestamp. Экономит часы.

Если хотите — скину в комменты.
```

## Важно

- Один маркер — не приговор. Иногда эмодзи уместен. Проблема — когда маркеры накапливаются.
- AI-score 3-4 — нормально. Не нужно доводить до 0, иначе потеряешь структуру.
- Цель — чтобы подписчик не подумал «опять AI-пост», а прочитал как авторский текст.
