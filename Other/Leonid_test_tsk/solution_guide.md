## 1.2. Какие расчётные показатели добавить

### A. Конверсии между шагами воронки (обязательно)

Добавить строки под каждой парой шагов:

| Метрика | Формула (пример для дня 1, столбец C) |
|---------|---------------------------------------|
| CTR офферов | `=C5/C3` (клик / офферы) |
| CR 1→2 (клик → экран подбора) | `=C6/C5` |
| CR 2→3 (подбор → к оформлению) | `=C7/C6` |
| CR 3→4 (к оформлению → финальный экран) | `=C8/C7` |
| CR 4→5 (финальный → получить деньги) | `=C9/C8` |
| CR 5→6 (получить → выдача) | `=C10/C9` |
| **CR total (клик → выдача)** | `=C10/C5` |

### B. Drop-off (потери на каждом шаге)

```
Drop 1→2 = 1 - CR 1→2
```
Показывает, на каком шаге теряем больше всего людей.

### C. Нарастающий итог выдач

```
Cumulative_day_N = SUM(C10:__10)   (от дня 1 до текущего)
```

### D. Динамика день-ко-дню

```
DoD_change = (Day_N - Day_N-1) / Day_N-1
```

### E. Средние/медианы за период

Для каждой метрики добавить справа: `AVERAGE()`, `MEDIAN()`, `STDEV()`.

## 2.3. Определение дефолта

**Стандарт Basel II/III: дефолт = DPD >= 90 дней.**

Создаём бинарную переменную:
```
default = IF(DPD >= 90, 1, 0)
Если DPD = null → default = 0
```

### Структура просрочки (DPD buckets)

| Бакет DPD | Количество | Доля от активных |
|-----------|-----------|-----------------|
| 0 (нет просрочки) | 19 962 | 73.2% |
| 1-30 дней | 1 228 | 4.5% |
| 31-60 дней | 551 | 2.0% |
| 61-90 дней | 524 | 1.9% |
| **90+ дней (дефолт)** | **5 017** | **18.4%** |

**Важное наблюдение:** бакеты 1-30 и 31-60 малочисленны, а 90+ — огромный. Это значит, что **клиенты, попавшие в просрочку, быстро "скатываются" в глубокий дефолт**. Работа с ранней просрочкой (soft collection на 1-30 DPD) критически важна — окно для возврата очень узкое.

## 2.6. Моделирование (что и как считать)

### Вариант A: Excel (если не знаешь Python)

**Шаг 1.** Создать колонку `default`:
```excel
=IF(ISNUMBER(G2), IF(G2>=90, 1, 0), 0)
```
где G2 — колонка «Дней просрочки».

**Шаг 2.** Создать вспомогательные колонки для группировки:
```excel
=IF(C2<=25,"18-25",IF(C2<=35,"26-35",IF(C2<=45,"36-45",IF(C2<=55,"46-55","56+"))))
=IF(B2<=100000,"до 100К",IF(B2<=250000,"100-250К",IF(B2<=500000,"250-500К","500К-1М")))
=IF(F2<=20,"0-20%",IF(F2<=40,"20-40%",IF(F2<=60,"40-60%",IF(F2<=80,"60-80%","80%+"))))
```

**Шаг 3.** Сводные таблицы (Pivot Tables):
- Строки: возрастная группа / сумма / регион
- Значения: COUNT(default), SUM(default), default rate = SUM/COUNT

**Шаг 4.** Корреляция:
```excel
=CORREL(default_range, age_range)
=CORREL(default_range, sum_range)
=CORREL(default_range, pdn_range)
```

**Шаг 5.** Надстройка «Анализ данных» → Регрессия:
- Y: default (0/1)
- X: сумма, возраст, пол (0/1), ПДН
- Результат: коэффициенты покажут вес каждого фактора

# Корреляционная матрица
corr_cols = ['amount','age','pdn','gender_num','default']

features = ['amount','age','pdn','gender_num']

## Правила оформления

### Общие

1. **Числа — с форматом.** Проценты через формат %, суммы с разделителем тысяч, рубли с ₽.
2. **Условное форматирование (conditional formatting).** Зелёный/красный для план-факт, gradient для дефолт-рейтов.
3. **Заголовки жирным, шрифт 12-14.** Данные — 10-11.
4. **Заморозить строки/столбцы** (Freeze Panes) для навигации.
5. **Именованные диапазоны** для формул (показывает профессионализм).
6. **Все формулы сохранить** — в задании прямо написано «важно сохранить расчеты и формулы».

### Графики

- Минимум декора: без 3D, без теней, без лишних сеток.
- Подписи осей + заголовок.
- Аномалии отметить: текстовая вставка или маркер.
- Цвета: 2-3 цвета максимум. Для дефолтов — красная палитра.

### Текст выводов

- **Структура:** наблюдение → причина → рекомендация.
- **Конкретные цифры:** «дефолт-рейт 18-25 лет = 15.6%, что в 1.9 раза выше 56+ (8.1%)», а не «молодые чаще дефолтят».
- **Бизнес-язык:** «рекомендуем ужесточить скоринг», а не «надо фильтровать молодых».
- **Приоритизация:** пронумеровать рекомендации по важности.

## Частые ошибки кандидатов (чего НЕ делать)

1. **Просто посчитать и не объяснить.** Таблица без выводов = 0 баллов. Нужны выводы текстом.

6. **Не сохранить формулы.** В задании явно написано — сохранить расчёты.
