# Content — YouTube + Telegram

## Контекст

Маркетинговый контент Mentor Analytics. Цель — привлечение учеников и формирование экспертности Алексея в теме аналитики и карьеры аналитика.

**Позиционирование:** «Аналитика без воды» — конкретно, практично, без лишних слов.

**Тематика строго:** аналитика, карьера аналитика, подготовка к собеседованиям, переход в профессию.
**Вне тематики:** лайфстайл, личная жизнь, мотивационный контент без пользы — не делаем, это размывает позиционирование.

## YouTube

### Типы видео

**Обучающее** — разбор темы (SQL, A/B тесты, продуктовые кейсы)
**Карьерное** — советы по поиску, резюме, собеседованиям
**Кейс/разбор** — реальный разбор резюме, кейса, собеса

### Workflow создания видео

**Шаг 1 → `brief.md`**

```markdown
# Бриф: [Название видео]

**Тип:** обучающее / карьерное / кейс
**Целевой запрос зрителя:** [что зритель ищет / какую проблему решает]
**Главный тезис:** [одно предложение — главная мысль видео]
**CTA:** [что зритель должен сделать после просмотра]
**Хронометраж:** ~X минут
**Нужен скринкаст?** да / нет

## Структура
1. Хук (0:00–0:30)
2. [Блок 1]
3. [Блок 2]
4. CTA + закрытие

## Статус
DRAFT / СОГЛАСОВАНО
```

**Шаг 2 → `script.md`** — только после согласования брифа.

### Правила сценария YouTube

**Хук (первые 30 сек) — критически важен:**
- Начинать с проблемы или неожиданного факта, не с представления
- Формат: «[Факт/проблема] → Вот что ты узнаешь»
- Пример: «90% резюме аналитиков отсеиваются на первом экране. Сейчас покажу почему и как это исправить за 20 минут.»

**Тело:**
- Один тезис = один блок
- Каждый блок: тезис → объяснение → пример → вывод
- Говорить как человек: короткие предложения, живые формулировки
- Без академизма и канцелярита

**Закрытие:**
- Резюме в 2-3 тезисах
- CTA: конкретный следующий шаг (ссылка, подписка, заявка)

### Формат файла сценария

```markdown
# Сценарий: [Название]

**Хронометраж:** ~X минут
**Бриф:** [ссылка на brief.md]

---

[00:00] **ХУК**
[Текст. Короткие предложения. Разговорный стиль.]

[00:30] **[Блок 1: Название]**
[Текст]

> 🎬 СКРИНКАСТ: [что показываем, если нужно]

[02:00] **[Блок 2: Название]**
[Текст]

[05:30] **ЗАКРЫТИЕ**
[Текст + CTA]

---

## Заметки для монтажа
- [Таймкоды для нарезки]
- [Где нужна графика]
- [Где можно убрать паузы]
```

## Telegram

### Типы постов

**Полезный** — практический совет, чек-лист, разбор ошибки
**Нарративный** — история (кейс ученика, личный опыт) с выводом
**Анонс** — анонс видео, гайда, набора

### Правила постов

**Длина:**
- Короткий (до 500 символов) — лайфхак, факт, вопрос
- Средний (500–1500) — разбор, история, чек-лист
- Длинный (1500+) — только если реально есть что сказать

**Структура среднего/длинного поста:**
1. Первые 1-2 строки = хук (виден до «читать далее»)
2. Тело — конкретика, примеры
3. Вывод или вопрос в конце

**Тон:** живой, прямой, без корпоративщины. Можно иронию, нельзя — занудство.

**Эмодзи:** умеренно, только по смыслу. Не в каждом предложении.

### Формат файла поста

```markdown
# Пост: [Тема]

**Тип:** полезный / нарративный / анонс
**Дата публикации:** YYYY-MM-DD

---

[Текст поста]

---

## Статус
DRAFT / ГОТОВО
```

## Пул идей

Файлы `youtube/ideas.md` и `telegram/ideas.md` — живые списки.

Формат строки:
```
- [Тип] [Название/тема] — [1 строка зачем] | Приоритет: высокий/средний/низкий
```

Пример:
```
- [YouTube/карьерное] «Почему не надо читать описание вакансии» — контраверсионный тезис, высокий retention | Приоритет: высокий
- [Telegram/полезный] Чек-лист перед публикацией резюме — привязка к гайду | Приоритет: средний
```

## Как начать работу над контентом

1. Прочитай этот CLAUDE.md
2. Прочитай корневой `../CLAUDE.md` для бизнес-контекста
3. Уточни задачу: новое видео, новый пост, или пополнить пул идей
4. Для видео: сначала `brief.md`, жди согласования, потом `script.md`
5. Для поста: можно сразу писать, но уточни тип и ключевой тезис
