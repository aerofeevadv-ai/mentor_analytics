# Content — YouTube + Telegram

## Контекст

Маркетинговый контент Mentor Analytics. Цель — привлечение учеников и формирование экспертности Алексея в теме аналитики и карьеры аналитика.

**Позиционирование:** «Аналитика без воды» — конкретно, практично, без лишних слов.

**Тематика строго:** аналитика, карьера аналитика, подготовка к собеседованиям, переход в профессию.
**Исключение:** IG и VK — 30-40% личного контента (фото, закулисье). Подробнее → `CONTENT_STRATEGY.md`, раздел 5.
**Юр. ограничения:** в VK-контенте запрещён логотип Instagram, прямые ссылки и призывы подписаться на IG. Подробнее → `CONTENT_STRATEGY.md`, раздел 5.1.

## YouTube

### Типы видео

**Обучающее** — разбор темы (SQL, A/B тесты, продуктовые кейсы)
**Карьерное** — советы по поиску, резюме, собеседованиям
**Кейс/разбор** — реальный разбор резюме, кейса, собеса

### Workflow создания видео

**Шаг 1 → `brief.md`**

```markdown
# Бриф: [Название видео]

**Тип:** обучающее / карьерное / кейс
**Целевой запрос зрителя:** [что зритель ищет / какую проблему решает]
**Главный тезис:** [одно предложение — главная мысль видео]
**CTA:** [что зритель должен сделать после просмотра]
**Хронометраж:** ~X минут
**Нужен скринкаст?** да / нет

## Структура
1. Хук (0:00–0:30)
2. [Блок 1]
3. [Блок 2]
4. CTA + закрытие

## Статус
DRAFT / СОГЛАСОВАНО
```

**Шаг 2 → `script.md`** — только после согласования брифа.

### Правила сценария YouTube

**Хук (первые 30 сек) — критически важен:**
- Начинать с проблемы или неожиданного факта, не с представления
- Формат: «[Факт/проблема] → Вот что ты узнаешь»
- Пример: «90% резюме аналитиков отсеиваются на первом экране. Сейчас покажу почему и как это исправить за 20 минут.»

**Тело:**
- Один тезис = один блок
- Каждый блок: тезис → объяснение → пример → вывод
- Говорить как человек: короткие предложения, живые формулировки
- Без академизма и канцелярита

**Закрытие:**
- Резюме в 2-3 тезисах
- CTA: конкретный следующий шаг (подписка, Telegram). НЕ предлагать менторство — продажа идёт только через Telegram, не из видео

### Формат файла сценария

```markdown
# Сценарий: [Название]

**Хронометраж:** ~X минут
**Бриф:** [ссылка на brief.md]

---

[00:00] **ХУК**
[Текст. Короткие предложения. Разговорный стиль.]

[00:30] **[Блок 1: Название]**
[Текст]

> 🎬 СКРИНКАСТ: [что показываем, если нужно]

[02:00] **[Блок 2: Название]**
[Текст]

[05:30] **ЗАКРЫТИЕ**
[Текст + CTA]

---

## Заметки для монтажа
- [Таймкоды для нарезки]
- [Где нужна графика]
- [Где можно убрать паузы]
```

## Telegram

### Типы постов

**Полезный** — практический совет, чек-лист, разбор ошибки
**Нарративный** — история (кейс ученика, личный опыт) с выводом
**Анонс** — анонс видео, гайда, набора

### Расписание

| День | Нечётная неделя | Чётная неделя |
|---|---|---|
| **Вт** | Кейс ученика | Разбор задачи с собеса |
| **Ср** | Полезный (совет, чек-лист, ошибка) | Полезный (мини-разбор, мнение) |
| **Чт** | Анонс YouTube-видео *(если вышло)* | Short + ссылка *(если есть)* |
| **Вс** | Личный инсайт | Лёгкий / ситуативный |

- Вт/Ср/Вс — стабильные, Claude пишет заранее на 2 недели
- Чт — привязан к YouTube, помощница публикует по факту выхода
- Если видео задерживается — четверг пропускается, остаётся 3 поста
- Разбор задач с собесов — реальные задачи от Алексея, адаптированные для поста

### Правила постов

**Длина:**
- Короткий (до 500 символов) — лайфхак, факт, вопрос
- Средний (500–1500) — разбор, история, чек-лист
- Длинный (1500+) — только если реально есть что сказать

**Структура среднего/длинного поста:**
1. Первые 1-2 строки = хук (виден до «читать далее»)
2. Тело — конкретика, примеры
3. Вывод или вопрос в конце

**Тон:** живой, прямой, без корпоративщины. Можно иронию, нельзя — занудство.

**Эмодзи:** умеренно, только по смыслу. Не в каждом предложении.

### Формат файла поста

```markdown
# Пост: [Тема]

**Тип:** полезный / нарративный / анонс
**Дата публикации:** YYYY-MM-DD

---

[Текст поста]

---

## Статус
DRAFT / ГОТОВО
```

## Пул идей

Файлы `youtube/ideas.md` и `telegram/ideas.md` — живые списки.

Формат строки:
```
- [Тип] [Название/тема] — [1 строка зачем] | Приоритет: высокий/средний/низкий
```

Пример:
```
- [YouTube/карьерное] «Почему не надо читать описание вакансии» — контраверсионный тезис, высокий retention | Приоритет: высокий
- [Telegram/полезный] Чек-лист перед публикацией резюме — привязка к гайду | Приоритет: средний
```

## Сценарии (запомнить только это)

Не нужно помнить отдельные скиллы. Просто скажи что хочешь сделать -- Claude подхватит нужный сценарий.

### 1. «Делаем видео» (или «делаем видео про [тему]»)

Что произойдёт:
1. Claude ищет конкурентов и транскрибирует лучшие видео по теме
2. Анализирует что они раскрывают и что пропускают
3. **Спрашивает тебя:** какие кейсы из опыта добавить, какую позицию занять
4. Готовит brief с вариантами хука (viral-формулы) -- на согласование
5. После ОК: пишет сценарий, ТЗ монтажёру, обложку (A/B), SEO, shorts
6. Проводит retention-ревью и скоринг

На выходе: полный пакет в `content/youtube/productions/SLUG/`

### 2. «Посты на 2 недели» (или «подготовь посты»)

Что произойдёт:
1. Claude определяет расписание на 2 недели (6 постов: Вт/Ср/Вс)
2. **Спрашивает тебя:** свежие кейсы учеников, задачи с собесов, личные инсайты
3. Генерирует все 6 постов пачкой
4. Каждый проходит AI-проверку и quality-проверку
5. Сохраняет в `content/telegram/posts/`

На выходе: 6 готовых постов, сводка с датами и scores

### 3. «Делаем курс по [теме]» (или «пишем раздел [N]»)

Что произойдёт:
1. Claude исследует тему: конкуренты, вопросы с собесов, пробелы
2. **Спрашивает тебя:** что реально спрашивают, какие кейсы включить
3. Готовит структуру -- на согласование
4. Готовит разделы тезисно -- на согласование
5. Пишет полный гайд + скрипты для видео
6. Проверяет качество (полнота, практичность, собесная готовность)

На выходе: полный курс в `courses/[тема]/`
