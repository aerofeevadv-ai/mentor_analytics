# 1.0 Введение: как работать с продуктовыми метриками

[🎬 ВИДЕО: говорящая голова — зачем этот модуль и как его проходить ~3 мин]

Эта страница задаёт рамку мышления для всего курса. Метрики мы начнём разбирать со следующей страницы — здесь фиксируем **как и зачем** ты будешь с ними работать.

---

## Почему смотреть на метрику недостаточно

В продуктовой аналитике ты постоянно видишь цифры: DAU, выручку, конверсию, retention. Но сами по себе они ничего не объясняют.

> 💡 **Главная мысль:** Одинаковый DAU у двух продуктов может означать совершенно разное состояние. Задача аналитика — не назвать метрику, а понять что за ней стоит.

---

## Продукт как система

Любой продукт — это система из трёх частей:

| Часть | Что сюда входит |
|-------|----------------|
| **Вход** | пользователи, трафик, установки, регистрации |
| **Поведение** | действия, сценарии, путь к ценности, возвращаемость |
| **Результат** | DAU, MAU, заказы, выручка, GMV |

Если смотреть только на результат — видишь **что произошло**, но не понимаешь **почему**.

---

## Два типа метрик

**Метрики результата** — DAU, MAU, выручка, GMV. Отвечают на вопрос: «Что получилось?»

**Метрики поведения** — конверсии, retention, воронки, частота действий. Отвечают на вопрос: «Почему результат такой?»

> 💡 **Главная мысль:** Метрики результата показывают симптом. Метрики поведения помогают найти причину.

**Пример:** DAU снизился. Это может значить: пришло меньше новых, старые стали реже возвращаться, сломался ключевой сценарий, изменился источник трафика. Одна метрика — разные причины — разные решения.

---

## Про формулы: учить обязательно

> ⚠️ **Типичная ошибка:** «Я понимаю логику, формулы не нужны». Без знания формул не работает декомпозиция метрик и теряется темп на собеседовании.

Рекомендация: выписывай метрики и формулы на листочек, повторяй утром и вечером. Не для зубрёжки — для того чтобы быстро раскладывать метрику на части в реальном времени.

---

## Инструмент не важен — важна логика

В этом курсе не принципиально где ты считаешь метрики.

```
ClickHouse  →  uniq(user_id)
SQL         →  count(distinct user_id)
pandas      →  .nunique()
```

Функция разная — смысл метрики один. Сначала понимаешь что считаешь, потом выбираешь инструмент.

## Что держать в голове по ходу курса

> 💡 **Главная мысль:** Рост метрики ≠ улучшение продукта. Одна метрика не даёт полной картины. Хороший аналитик сначала задаёт вопросы, потом считает.

## 🏪 Сквозной кейс — Маркетплейс «М»

На протяжении всего модуля мы будем возвращаться к одному и тому же продукту — вымышленному маркетплейсу товаров повседневного спроса (аналог Ozon / Wildberries). Это поможет увидеть, как метрики из разных тем связаны между собой на одном продукте.

**Профиль «М»:**

| Метрика | Значение |
|---------|----------|
| MAU | 800 000 |
| DAU | 120 000 |
| Новых пользователей/день | 3 000 |
| Retention D1 / D7 / D30 / плато | 45% / 25% / 15% / 12% |
| Воронка (день): Приложение → Каталог → Карточка → Корзина → Заказ | 120к → 84к → 48к → 14.4к → 8.6к |
| Заказов/мес | ~260 000 |
| AOV | 2 000 ₽ |
| GMV/мес | 520 000 000 ₽ |
| Take Rate | 15% |
| Revenue/мес | 78 000 000 ₽ |
| Платящих/мес | 150 000 |
| ARPU | 97 ₽ |
| ARPPU | 520 ₽ |
| Маркетинг/мес | 18 000 000 ₽ |
| CAC | 720 ₽ |
| Churn | 8%/мес |
| LTV | 6 500 ₽ |
| LTV/CAC | ~9 |

Не нужно запоминать все числа сейчас — каждую из этих метрик мы подробно разберём в модуле. Сейчас достаточно увидеть общую картину: это живой продукт со своими метриками, а не набор абстрактных формул. По ходу курса ты будешь возвращаться к этой таблице и замечать, как числа связаны между собой.
