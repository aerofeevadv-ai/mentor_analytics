|---|------|-------|----------------|

> 💡 **Отличие от мидл-собеса:** кейс не про «разложи метрику» или «проверь гипотезу», а про **«построй метрику, которой не существует»**. Это сеньорный уровень — умение работать в полной неопределённости.

## Кейс: как измерить сделки на Авито?

### Контекст задачи

Авито — маркетплейс объявлений. Покупатель находит товар → связывается с продавцом → сделка происходит **вне платформы** (по телефону, при встрече). Авито не видит, состоялась ли сделка.

**Задача:** придумать способ оценивать, сколько контактов покупатель–продавец реально заканчиваются сделкой.

**Зачем бизнесу:** понимать GMV (Gross Merchandise Value), оценивать эффективность монетизации, находить точки роста конверсии.

---

### Этап 1. Понимание бизнеса

Кандидат начал с правильного вопроса: «Как монетизируется Авито?»

**Модель монетизации:**
- Платные размещения и продвижение объявлений
- Пакеты для профессиональных продавцов
- Реклама

**Ключевой инсайт:** если мы не знаем, какие контакты заканчиваются сделкой — мы не можем оптимизировать конверсию и честно считать GMV.

### Этап 2. Какие данные есть?

Кандидат прошёлся по CJM (Customer Journey Map) покупателя:

```
Поиск → Просмотр объявления → Контакт (звонок/чат) → [???] → Сделка
```

**Доступные сигналы:**
| Источник | Что можно извлечь |
|----------|------------------|
| Звонки (через подменный номер) | Факт звонка, длительность, запись разговора |
| Чат в приложении | Текст переписки, ключевые слова |
| Поведение продавца | Снятие объявления, архивация |
| Объявление | Категория, цена, время жизни |
| История пользователя | Прошлые контакты, рейтинг продавца |

### Этап 3. Определение «целевого контакта»

> ⚠️ **Нюанс:** «контакт ≠ сделка». Человек может позвонить, спросить цену и передумать. Нужно отделить «пустые» контакты от «целевых».

**Сигналы целевого контакта:**
- В разговоре обсуждают адрес, время встречи, обмен контактами
- Длительность звонка выше порога (для категории)
- После контакта объявление снимается в течение N дней

**Важно:** сигналы различаются по категориям:
- **Телефоны** — быстрая сделка, снятие объявления через 1–2 дня
- **Недвижимость** — долгий цикл, обсуждение адреса и показа
- **Услуги** — сделка может быть разовой, объявление не снимается

### Этап 4. Разметка данных (labeling)

Как получить ground truth — «была сделка или нет»?

**Подход 1: Прозвон продавцов**
- Берём выборку контактов, звоним продавцу: «Вы продали товар?»
- Плюс: точные данные
- Минус: дорого, не масштабируется

**Подход 2: Бизнес-правила**
- Если позитивные сигналы в переписке + объявление снято в течение 7 дней → «сделка»
- Плюс: масштабируемо
- Минус: много ложных срабатываний

**Решение:** гибрид — прозвон для обучающей выборки, бизнес-правила для масштабирования.

### Этап 5. Подход к модели

Кандидат предложил разумную последовательность:

```
1. Простые правила (regex + фильтры) → baseline
2. Оценить: хватает ли точности?
3. Если нет → ML (логистическая регрессия, градиентный бустинг)
```

**Фичи для модели:**
| Группа | Примеры |
|--------|---------|
| Объявление | Категория, цена, время жизни, рейтинг продавца |
| Поведение покупателя | История контактов, клики, прошлые сделки |
| Контент взаимодействия | Ключевые слова из чата/звонка |
| Сигналы звонка | Длительность, наличие конфликтных маркеров |

### Этап 6. Качество модели

> ⚠️ **Ловушка:** данные несбалансированы — сделок значительно меньше, чем «пустых» контактов. Accuracy врёт.

**Правильные метрики:**
| Метрика | Что показывает |
|---------|---------------|
| Precision | % предсказанных сделок, которые реально были сделками |
| Recall | % реальных сделок, которые мы нашли |
| F1-score | Баланс precision и recall |

**Перевод для бизнеса:** «Мы находим X% реальных сделок (recall), и Y% наших предсказаний верны (precision).»

### Этап 7. Мониторинг и data drift

Модель со временем деградирует:
- Пользователи меняют поведение
- Появляется новый сленг в переписках
- Меняется ассортимент категорий

**Как детектировать дрифт:**
- Сравнивать распределения фичей по временным окнам (mean, std, квантили)
- Статистические тесты: **Колмогоров-Смирнов** (сдвиг распределения), **Манн-Уитни** (сдвиг рангов)
- Периодическая ревалидация: сравнение предсказаний с прозвоном

**Переобучение:** еженедельно (не ежедневно — нужно время, чтобы увидеть исход контакта).

## Чем этот собес отличается от мидл-уровня

| Аспект | Mid+ (3.2) | Senior (3.3) |
|--------|-----------|-------------|
| Тип кейса | Разложи метрику, предложи гипотезы | Построй метрику с нуля |
| Данные | Даны или очевидны | Нужно самому определить, что доступно |
| Неопределённость | Средняя — есть рамки | Высокая — нет правильного ответа |
| ML/статистика | CUPED, стратификация | Data drift, precision/recall, labeling |
| Бизнес-мышление | Связать метрику с продуктом | Связать метрику с монетизацией и GMV |
