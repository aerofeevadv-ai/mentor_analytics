## Блок 2. Behavioral — что спрашивают

Авито оценивает по **публичной матрице компетенций** (есть на GitHub и Хабре):

| Компетенция | Вопрос |
|-------------|--------|
| Поиск задач | «Как к тебе попадает задача? Откуда они берутся?» |
| Планирование | «Как устроено планирование? Какой горизонт?» |
| Неопределённость | «Задача с наибольшей степенью неопределённости в формулировке?» |
| Сложность | «Пример задачи, где применял сложную аналитику или математику?» |
| Приоритизация | «Несколько заказчиков — как разруливал?» |

> 💡 **Инсайт Mid vs Senior:** для Mid достаточно реактивного подхода («пришли — сделал»). Для Senior — ожидается проактивность («сам нашёл точку роста, принёс данные»).

## Блок 3. Техзадачка — кубик

**Условие:** игральный кубик с числами 1–6. Бросаешь и забираешь столько долларов. Можешь перебросить один раз. Оптимальная стратегия? Ожидаемый выигрыш?

<details>
<summary>Разбор</summary>

**Стратегия:** перебрасываешь, если выпало ≤ 3. Оставляешь, если 4, 5, 6.

**Почему:** матожидание кубика = 3.5. Если выпало ≤ 3 — ожидание переброса (3.5) выше текущего. Если выпало ≥ 4 — текущее лучше.

**Расчёт:**
- Вероятность ≤ 3: 3/6 = 50% → перебрасываем, EV = 3.5
- Вероятность ≥ 4: 3/6 = 50% → оставляем, EV = (4+5+6)/3 = 5

**Общее EV:** 0.5 × 3.5 + 0.5 × 5 = **4.25**

</details>

## Блок 4. Продуктовый кейс — Telegram

Кандидат выбрал Telegram. Интервьюер ведёт по этапам:

### Этап 1. Формулировка проблемы

Кандидат: «В групповых чатах неудобно искать конкретное сообщение.»

Интервьюер: «Как аналитик, как ты эту долю чатов считал бы?»

> 💡 **Хороший приём:** оценка сверху и снизу. Сверху: MAU × % пользователей групп × % использующих поиск. Снизу: логи поисковых запросов в чатах.

### Этап 2. Валидация — стоит ли решать?

Ключевой вопрос: затрагивает ли проблема достаточно пользователей, чтобы влиять на MAU/retention?

### Этап 3. Выбор метрик

Кандидат предложил «количество отправленных сообщений».

> ⚠️ **Фидбек интервьюера:** метрика не связана с исходной проблемой. Если проблема — поиск в чатах, метрика должна отражать поиск (доля успешных поисков, retention после использования поиска).

### Этап 4. Дизайн A/B-теста

Обсуждение:
- Сетевые эффекты: пользователи из разных групп общаются — рандомизация по пользователям может быть проблемой
- «Серые» результаты: что делать, если метрика не значима?
- Методы снижения дисперсии: CUPED, стратификация

## Ключевые уроки

| Что сделал хорошо | Что можно лучше |
|-------------------|----------------|
| Подготовил рассказ по STAR | Ключевая метрика не связана с проблемой |
| Сам упомянул сетевые эффекты | Не назвал контр-метрики |
| Оценка сверху и снизу | Задача пришла от продакта, а не найдена самостоятельно |
