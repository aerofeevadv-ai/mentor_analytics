# 3.3 Где встречаются продуктовые кейсы и как готовиться

---

## Где дают продуктовые кейсы

### Т-Банк

- **Отдельная продуктовая секция** — идёт сразу после технической. Полный формат из 7 блоков (как в [3.1](3_1_tbank_case.md)): бизнес-модель → стадия → конкуренты → фичи → RICE → дашборды → A/B
- **Финалы с командами** — когда встречаешься с конкретной командой, могут дать продуктовые вопросы уже по их зоне ответственности

### Авито

- **Отдельная продуктовая секция** — подробный разбор паттернов в [3.2](3_2_avito_mid.md)
- **Финалы** — продуктовые вопросы по конкретному юниту/команде
- Особенность: на behavioral-части важно показать **проактивность** — ты сам ставил себе задачи, а не только выполнял

### Яндекс

Всё зависит от направления, команды и юнита, но общий паттерн:
- **Продуктовая секция** — обычно есть для продуктового аналитика. Могут дать 1-2 кейса: сначала небольшую задачку, потом более крупную. Формат похож на банк вопросов из [3.4](3_4_question_bank.md): придумай метрику, разложи проблему, предложи эксперимент
- **Финалы** — продуктовые вопросы **по конкретной команде**. Это ключевое отличие: на секции спрашивают общие кейсы, на финале — про конкретный юнит, чем он занимается, какие вызовы

### Другие компании

**Бигтехи (2+ этапа + финал):**
- Часто продуктовая секция — отдельный этап
- На финалах тоже дают продуктовые кейсы — почти всегда

**Средние и небольшие компании (1-2 этапа):**
- Часто второй этап = финал, и на финале приходит продакт-менеджер с продуктовыми кейсами
- Иногда продуктовый кейс дают прямо на техническом собесе (после Python/SQL)

> 💡 **Если свести к общему:** в бигтехах продуктовая секция отдельно + на финалах снова продуктовые кейсы. В небольших компаниях — обычно на финале.

## Как проходить продуктовые собесы: чек-лист подготовки

### Перед выходом на собесы вообще

Разложи самостоятельно **несколько бизнес-моделей** на составляющие и имей перед глазами:

| Тип бизнеса | Пример | На что обратить внимание |
|-------------|--------|--------------------------|
| E-commerce / маркетплейс | Ozon, Wildberries | GMV, Take Rate, unit-экономика логистики |
| Банк | Т-Банк, Сбер | Interchange fee, кредитная маржа, ARPU по продуктам |
| Такси / агрегатор | Яндекс Go | Двусторонний маркетплейс, баланс водитель/пассажир |
| Мессенджер / соцсеть | Telegram, VK | Рекламная модель, DAU/MAU, engagement |
| Подписка | Яндекс Плюс, Spotify | MRR, churn, LTV, expansion revenue |

Для каждого:
- [ ] Как зарабатывает
- [ ] Дерево метрик по фреймворку
- [ ] North Star Metric + обоснование
- [ ] Ключевые вызовы и конкуренты

### Перед конкретной компанией (продуктовая секция)

- [ ] Понять бизнес-модель компании
- [ ] Посмотреть конкурентов (три типа)
- [ ] **Разложить дерево метрик** — NSM → Revenue → конверсии → retention
- [ ] Продумать **North Star Metric** с полной иерархией и уметь объяснить, почему именно она
- [ ] Если есть кейсы из этого курса — подготовиться по ним
- [ ] Если нет — попрактиковаться с AI (промпт ниже)

### Перед финалом в конкретную команду

- [ ] Узнать, **чем конкретно занимается команда** — какой юнит, какой продукт
- [ ] Понять **вызовы команды** — что сейчас растят, что болит, какие метрики в фокусе
- [ ] Подготовить продуктовые кейсы **по этому юниту** — вероятнее всего, вопросы будут именно про него
- [ ] Иметь мнение: какие фичи бы ты предложил для этой команды

> 💡 **Паттерн:** на продуктовых секциях — общие кейсы. На финалах — узкоспециализированные, по конкретной команде.

## Промпт для подготовки с AI

Скопируй и адаптируй под свою ситуацию:

```
Я иду на продуктовую секцию собеседования в [компания] на позицию продуктового аналитика.

1. Расскажи, как [компания] зарабатывает. Перечисли основные потоки дохода.
2. Кто конкуренты — прямые, непрямые, потенциальные?
3. Помоги мне разложить дерево метрик для [компания]: от North Star Metric до операционных метрик.
4. Какие ключевые вызовы стоят перед [компания] сейчас?
5. Задай мне 3 продуктовых кейса по [компания] и проверь мои ответы. Давай по одному.
```

Для финала в конкретную команду:

```
Я иду на финальное собеседование в [компания], команда [название / описание].
Эта команда занимается [что ты знаешь о команде].

1. Какие метрики скорее всего в фокусе этой команды?
2. Какие вызовы могут стоять перед ней?
3. Задай мне продуктовый кейс, который мог бы дать интервьюер из этой команды.
```

## Как проходить: фреймворк для любого кейса

Независимо от компании, продуктовый кейс проходится по одной структуре:

```
1. Понять бизнес → Как зарабатывает? Кто конкуренты?
2. Определить проблему → Какая боль? Кого затрагивает?
3. Предложить решение → Фича, механизм, метрика
4. Оценить потенциал → Feature sizing, RICE
5. Спроектировать проверку → A/B-тест, guardrails
6. Продумать дашборд → Кому, для чего, какие метрики
```

Этот фреймворк — суть модулей 1–3 этого курса.
