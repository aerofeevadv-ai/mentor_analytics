# 2.2 Кейс: упала метрика

Самый частый тип кейса на собеседованиях. Интервьюер называет метрику и говорит, что она изменилась. Твоя задача — найти причину.

---

## Алгоритм (КМДГП)

1. Уточни контекст (какая метрика, насколько, за какой период, какой сегмент)
2. Зафиксируй формулу метрики
3. Разложи на дерево — проверь каждую ветку
4. Сформулируй гипотезы для ветки, которая изменилась
5. Предложи проверку

---

## Практика: скорость загрузки Морды Яндекса

> Источник: реальный кейс с собеседования в Яндексе.

**Условие:** В течение суток собирали данные о скорости загрузки страницы yandex.ru для каждого хита. График распределения показывает два горба — основной и второй, правее и ниже.

> 🎯 **Практика:** Почему на графике два горба? Сгенерируй гипотезы, прежде чем читать разбор.

---

### Разбор

**Шаг 1. Что мы видим:**
- Распределение начинается правее нуля (страница не загружается мгновенно)
- Основной горб — нормальная скорость загрузки
- Второй горб правее — группа хитов с заметно более медленной загрузкой
- Два горба = **сумма двух распределений** → есть фактор, который бинарно влияет на скорость

**Шаг 2. Гипотезы:**

| Гипотеза | Почему может быть верна |
|----------|----------------------|
| Ошибка в данных | Часть серверов или браузеров записывают данные с множителем или смещением |
| Мобильная версия | Мобильная Морда загружается медленнее десктопной — два отдельных сегмента |
| Авторизованные пользователи | Почтовая шапка у авторизованных тормозит загрузку |
| Кастомные виджеты | Пользователи с настроенной Мордой (виджеты, скины) грузятся дольше |
| Первая загрузка vs кеш | Первый визит загружает статику, повторный — из кеша |
| Авария в течение дня | Часть суток серверы тормозили → отдельный горб |
| Пиковая нагрузка | При превышении порога rps время загрузки растёт нелинейно — скачком |
| Один тормозящий сервер | Если один из серверов медленнее — его хиты формируют второй горб |

**Шаг 3. Как проверить:**
- Разрезать по устройству (мобайл / десктоп)
- Разрезать по авторизации (да / нет)
- Разрезать по времени суток
- Посмотреть распределение по серверам

> 💡 **Ключевой инсайт:** Два горба — это **два сегмента**. Задача аналитика — найти фактор, который разделяет аудиторию на две группы.

---

## Практика: DAU вырос, а выручка — нет

**Условие:** DAU вырос на 20% за месяц, Revenue остался без изменений.

> 🎯 **Практика:** Разложи по КМДГП. Где искать проблему?

<details>
<summary>Разбор</summary>

**Д (декомпозиция):**
```
Revenue = Платящие × ARPPU
Платящие = DAU × Конверсия в покупку
```

DAU вырос, Revenue нет → значит, конверсия в покупку или ARPPU упала.

**Г (гипотезы):**
1. Пришли «пустые» пользователи из рекламы — DAU ↑, но конверсия ↓
2. Изменился микс аудитории: новые пользователи с более низким чеком
3. Промоакция привлекла трафик, но не покупателей

**П (проверка):**
- Конверсия в покупку по когортам: старые vs новые
- ARPPU по источникам трафика
- Каналы привлечения новых пользователей за этот месяц

</details>
