# 4.1 Кейс: упала конверсия в доставке еды

**Тип:** упала метрика
**Время:** 15–20 минут
**Что тренируешь:** КМДГП, декомпозиция, генерация гипотез

---

## Условие

Ты аналитик в сервисе доставки еды (аналог Delivery Club / Яндекс Еда). Продакт-менеджер пишет в чат:

> «Конверсия из корзины в оплаченный заказ упала на 15% за последнюю неделю. Что происходит?»

У тебя есть доступ к:
- Логам событий (просмотр ресторана → добавление в корзину → оформление → оплата → доставка)
- Сегментам: платформа (iOS/Android/web), город, новые/вернувшиеся, способ оплаты
- Истории релизов приложения
- Данным о ресторанах (рейтинг, время приготовления, зона доставки)

**Задание:** разбери кейс по структуре. Проговори решение вслух, как на собеседовании. Засеки время.

---

<details>
<summary>💡 Подсказки (открой, если застрял)</summary>

1. Начни с уточнения: какая именно конверсия? Из корзины в оплату — это один шаг или несколько?
2. Разложи воронку: корзина → оформление → выбор оплаты → подтверждение → оплачено
3. Проверь разрезы: платформа, город, новые vs вернувшиеся
4. Подумай о внешних факторах: не менялся ли минимальный заказ? Не было ли релиза?

</details>

---

<details>
<summary>📝 Полный разбор</summary>

### К — Контекст

- **Метрика:** CR (корзина → оплаченный заказ)
- **Период:** неделя, значит сравниваем week-over-week
- **Масштаб:** −15% — вероятно, существенное падение, но стоит проверить: какая обычная дисперсия у этой метрики? Если week-over-week колебания обычно ±2-3%, то −15% — точно сигнал. Если дисперсия высокая (±10-15%) — нужно смотреть на более длинном горизонте

**Уточняющие вопросы:**
- Это все пользователи или конкретный сегмент?
- Абсолютное число заказов тоже упало, или только конверсия?
- Были ли релизы на этой неделе?

### М — Метрика

Разложим воронку на шаги:

```
Корзина → Оформление → Выбор оплаты → Подтверждение → Оплата прошла
```

На каком шаге падение? Это сужает гипотезы:
- Корзина → оформление: проблема с UX или минимальным заказом
- Выбор оплаты → подтверждение: проблема с платёжным методом
- Подтверждение → оплата: технический сбой

### Д — Декомпозиция

| Разрез | Что проверяем |
|--------|--------------|
| Платформа | iOS vs Android vs web — может падение только на одной |
| Город | Москва vs регионы — может проблема локальная |
| Новые / вернувшиеся | Если только у новых — проблема онбординга |
| Способ оплаты | Карта vs Apple Pay vs наличные — может сломался один метод |
| Тип ресторана | Сетевые vs несетевые — может изменились условия |

### Г — Гипотезы

**Технические:**
1. Баг после релиза на одной платформе (например, кнопка «оплатить» не работает на Android 14)
2. Сбой платёжного шлюза для определённого банка
3. Увеличилось время загрузки страницы оплаты → пользователи уходят
4. **Проблемы с данными:** события не записываются корректно, данные не подтянулись из одного источника, дубли или потери в логировании — конверсия «упала» в данных, но не в реальности

**Продуктовые:**
4. Повысили минимальную сумму заказа → часть корзин не проходит
5. Убрали промокод или акцию → цена стала выше ожидаемой
6. Изменили стоимость доставки → пользователь видит итог и отказывается

**Внешние:**
7. Конкурент запустил агрессивную акцию → пользователи сравнивают и уходят
8. Зарплатная неделя / конец месяца — сезонный эффект (но −15% много для сезона)

### П — Проверка

| Гипотеза | Как проверить |
|----------|--------------|
| Баг на Android | CR по платформам, лог ошибок, дата релиза |
| Сбой оплаты | Доля ошибок оплаты по провайдерам |
| Мин. заказ | Чейнджлог бизнес-правил, CR до/после |
| Убрали акцию | Доля заказов с промокодом за 2 недели |
| Стоимость доставки | Средний чек с доставкой, drop-off на шаге «итого» |
| Проблемы с данными | Объём событий по дням — резкое падение = потеря логов |

**Контр-метрики:** общее число заказов, средний чек, retention D7, DAU — чтобы убедиться, что проблема именно в конверсии, а не в трафике.

**Вывод:** скорее всего, причина техническая (баг или сбой оплаты) или продуктовая (изменение ценообразования). Начинаю с проверки по платформам и лога ошибок — это самый быстрый способ сузить.

</details>

<details>
<summary>🎯 Критерии оценки</summary>

| Что оценивается | Хорошо | Плохо |
|----------------|--------|-------|
| Уточнил контекст | Спросил про период, сегмент, абсолютные числа | Сразу начал гипотезы |
| Разложил воронку | По шагам, нашёл где именно drop-off | «Конверсия упала» — и всё |
| Декомпозиция | ≥3 разреза, объяснил зачем каждый | Один разрез или хаотично |
| Гипотезы | Разной природы (тех + продукт + внешний), ≥5 | Все одного типа, <3 |
| Проверка | Конкретный способ для каждой гипотезы | «Посмотрю в данных» без деталей |
| Контр-метрики | Назвал 2–3 | Не упомянул |
| Структура | Двигался по КМДГП | Прыгал хаотично |

</details>
