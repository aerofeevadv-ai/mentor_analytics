# Courses 20/80 — Контекст и workflow

## Что это

Структурные гайды по темам аналитики: 20% информации → 80% пользы для работы и собеседований.

**Аудитория:**
- Первичная: ученики менторства «до оффера» (доступ включён)
- Вторичная: внешние покупатели (отдельные гайды или весь road map)

**Формат:** текст + видео. Текст — основа, видео — усиление в ключевых местах.

## Трёхшаговый workflow создания курса

### Шаг 1 → `structure.md`
Согласовать структуру и основные разделы.

Файл: `[тема]/structure.md`

Формат:
```markdown
# [Название курса] — Структура

## Цель курса
[Что ученик умеет после прохождения]

## Разделы
1. [Название раздела] — [1-2 строки о чём]
2. ...

## Объём
[Примерное количество страниц / минут видео]
```

**Статус:** DRAFT → СОГЛАСОВАНО (вручную Алексеем)

---

### Шаг 2 → `sections/XX_название.md`
Для каждого раздела согласовать содержание — тезисно, без полного текста.

Файл: `[тема]/sections/01_название.md`

Формат:
```markdown
# Раздел 1: [Название]

## Ключевые тезисы
- [Что будет объяснено]
- ...

## Практика
- [Задачи / упражнения]

## Видео нужно? 
- [ ] Нет
- [ ] Да — говорящая голова: [тема]
- [ ] Да — скринкаст: [что показываем]

## Статус
DRAFT / СОГЛАСОВАНО
```

---

### Шаг 3 → `guide.md` + `scripts/`
После согласования всех разделов — Claude пишет полный гайд и сценарии.

**Файлы:**
- `[тема]/guide.md` — финальный гайд в Notion-совместимом Markdown
- `[тема]/scripts/intro_script.md` — сценарий онбординг-видео (всегда)
- `[тема]/scripts/screencast_NN.md` — сценарии скринкастов (по необходимости)

---

## Правила написания гайда

**Тон:** практичный, конкретный, без воды. «Аналитика без воды»: 20% теории, 80% действий и примеров.

**Структура каждого гайда:**
1. Онбординг-блок (как проходить, что получишь)
2. Разделы по структуре
3. Чек-лист в конце (что должен уметь после)

**Форматирование (Notion-совместимый Markdown):**
- H1 — название курса (один раз)
- H2 — разделы
- H3 — подразделы
- Таблицы, code blocks, callout-блоки (`> 💡 ...`)
- Эмодзи для навигации: 📌 важное, ⚠️ типичная ошибка, ✅ пример, 🎯 практика

**Длина:** насыщенно, не раздуто. Лучше короче и с примерами, чем длинно и абстрактно.

## Правила про видео

Claude сам определяет где нужно видео по следующим критериям:

**Говорящая голова (онбординг):**
- Всегда в начале каждого курса — одно обязательное онбординг-видео
- Алексей объясняет: зачем этот курс, как проходить, что будет в конце

**Скринкаст нужен если:**
- Показываем работу в инструменте (Jupyter, SQL-редактор, Tableau)
- Объясняем код построчно (сложнее одного экрана)
- Разбираем кейс пошагово с данными
- Демонстрируем дашборд или визуализацию

**Скринкаст НЕ нужен если:**
- Концепция объясняется текстом + примером
- Формула или таблица достаточно наглядна
- Раздел короче 5 минут устного объяснения

## Шаблон сценария для телесуфлёра

Файл: `scripts/intro_script.md` или `scripts/screencast_NN.md`

```markdown
# Сценарий: [Название видео]

**Тип:** говорящая голова / скринкаст
**Хронометраж:** ~X минут
**Раздел курса:** [к какому разделу относится]

---

## До записи
- [ ] Открыть: [что должно быть на экране, если скринкаст]
- [ ] Проверить: [технические детали]

---

## Сценарий

[00:00] **Вступление**
[Текст для телесуфлёра. Каждый абзац — отдельный смысловой блок. 
Короткие предложения. Говорить как человек, не читать как диктор.]

[01:30] **Основная часть**
[...]

[04:00] **Закрытие**
[...]

---

## Заметки для монтажа
- [Что вырезать / добавить]
- [Где нужна графика или анимация]
```

## Как начать новый курс

1. Прочитай этот CLAUDE.md
2. Прочитай корневой `../CLAUDE.md` для бизнес-контекста
3. Спроси Алексея: «Начинаем с какого курса?»
4. Создай `[тема]/structure.md` и предложи структуру
5. Жди подтверждения перед переходом к Шагу 2

**Не пиши guide.md пока structure.md и все sections/ не получили статус СОГЛАСОВАНО.**
