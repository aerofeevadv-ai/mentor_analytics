### Часть A. Фундамент

**Введение**
- Pandas = «Excel на стероидах»: DataFrame — таблица с методами, Series — колонка
- Зачем фундамент до методов: иначе методы воспринимаются как магия без логики

**DataFrame и Series**
- Отличия (таблица 2D/1D), индекс, выравнивание по индексу
- Создание DataFrame: из словаря списков, списка словарей, списка списков

**Загрузка данных**
- read_csv: базовая + ключевые параметры sep (";" в российских CSV), encoding (utf-8/cp1251)
- Дополнительно: nrows, usecols
- Частые ошибки: FileNotFoundError, UnicodeDecodeError, ParserError — причина и решение
- Excel (openpyxl), JSON (lines=True), URL
- SQL — концептуально: в компании ты пишешь SQL → получаешь DataFrame → анализируешь
  в pandas. SQL для выборки, pandas для обработки. Тебе это уже знакомо

**Первичный анализ**
- head / tail / sample / info / describe / dtypes / shape
- «Порядок профи»: shape → head → info → describe → sample, 20–30 секунд на таблицу

**Базовые операции**
- Колонки: выбор одной/нескольких, создание, drop, rename
- Базовая фильтрация: булева маска как концепция (df["age"] > 30 → True/False)
- Индекс: метка ≠ позиция, index / reset_index / set_index
- Пропуски: NaN/None, isna().sum(), почему ломают вычисления
- Типы: int / float / object / bool / datetime64
- Векторные операции: df["price"] * 1.2 — не цикл; «pandas думает массивами»

### Часть B. Ключевые методы 20/80 — 10 групп карточками

Формат карточки метода (стандарт): **Что делает / Простой пример (80% случаев) /
Чуть сложнее / Подводные камни / Аналог в SQL / Доп. параметры (спойлер)**

| # | Группа | Методы | Аналог в SQL |
|---|---|---|---|
| 1 | Просмотр | head, tail, sample, info, describe, dtypes, shape | LIMIT |
| 2 | Фильтрация | маски, & \| ~, isin, between*, query | WHERE, IN, BETWEEN |
| 3 | Колонки | rename, astype, replace, value_counts, apply*, map* | AS, CAST, CASE |
| 4 | Группировки | groupby, agg, nunique, pivot_table | GROUP BY, COUNT DISTINCT |
| 5 | Объединение | merge, concat | JOIN, UNION |
| 6 | Пропуски | isna, notna, fillna, dropna | IS NULL, COALESCE |
| 7 | Дубликаты | duplicated, drop_duplicates | DISTINCT |
| 8 | Сортировка | sort_values | ORDER BY |
| 9 | Текст | str.contains, str.lower/upper, str.strip | LIKE, LOWER/UPPER |
| 10 | Чтение/запись | read_csv, to_csv, read_excel*, to_excel* | — |

- SQL-параллель в каждой карточке — главный дидактический рычаг для нашей аудитории
- Группы 1–2 уже написаны в драфте, 3–10 пишем по тому же шаблону
- Подводные камни обязательны: скобки в составных условиях, merge с дублями ключа,
  groupby теряет NaN, value_counts(dropna), chained assignment
- Группа 11 (опционально, со звёздочкой): apply/map глубже, validate в merge, suffixes
