# Модуль 5: Алгоритмы для собеседований (расширенный тариф)

## Ключевые тезисы

Цель: покрыть ~90% алго-вопросов на собесах аналитиков. Без структур данных
(деревья, графы, DP) — этого аналитикам не дают.

**Зачем и где спрашивают**
- ~20% собесов, чаще бигтех (особенно Яндекс)
- Уровень: easy / лёгкий medium LeetCode
- В работе почти не нужно, но отсев на собесе реальный

**Сложность алгоритмов: Big-O на пальцах**
- Без математики: O(1) «сразу», O(n) «один проход», O(n²) «цикл в цикле»,
  O(n log n) «сортировка»
- Как отвечать на вопрос «какая сложность у твоего решения?» — типовой follow-up
- Почему dict/set ищут за O(1), а список за O(n) — и как это использовать

**Приём 1. Проход по массиву (один цикл)**
- max/min/сумма без встроенных функций
- Частоты через dict (ручной Counter)
- Уникальные через set
- Второй максимум, подсчёт пар
- Шаблон: аккумулятор → цикл → обновление условия

**Приём 2. Два указателя**
- Концепция: когда два индекса убирают вложенный цикл (O(n²) → O(n))
- Пара с заданной суммой в отсортированном массиве
- Палиндром
- Слияние двух отсортированных списков
- Разворот списка на месте

**Формат решения на собесе**
- Всегда функция solve(args) с return
- Алгоритм поведения: уточни условие → edge cases (пустой список, один элемент,
  дубли, отрицательные) → скажи brute force → улучшай → проговаривай вслух
- Что делать, если решение не приходит: честный brute force лучше молчания

## Практика

- Тренировочный набор 15–20 задач, 3 уровня:
  - разогрев (5–6): проход по массиву
  - основа (6–8): частоты, set, второй максимум, пары
  - два указателя (4–6): сумма, палиндром, слияние
- Каждая задача: условие → подсказка (спойлер) → решение с построчным разбором → сложность
- Методика: решать руками без LLM, повторить набор дважды
