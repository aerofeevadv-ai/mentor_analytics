# Модуль 3: Практика до автоматизма

## Ключевые тезисы

Ядро курса. Всё новое здесь — задачи и датасеты, теории почти нет.

**Методология автоматизации (вводная страница)**
- Навык = повторение: один и тот же набор задач решается 5–7 дней подряд
- Критерий автоматизма: 20 задач ежедневного практикума за 25–30 минут без подсказок
- Прогресс-трекер: таблица в Notion (день × время × ошибки), ученик заполняет сам

**Блок 1. Ежедневный практикум: 20 задач**
- Покрывают все 10 групп методов модуля 2 (по 2 задачи на группу)
- Один датасет, фиксированный порядок, известные ответы для самопроверки
- Решения в спойлерах + эталонное время на задачу

**Блок 2. Тематические датасеты (5 штук, синтетика)**
Генерим скриптами (ops-скрипт в репо, ученикам — готовые CSV).
В каждом датасете контролируемая «грязь»: пропуски, дубли, числа строками, кривые даты.

| Датасет | Таблицы | Особенность | Акцент задач |
|---|---|---|---|
| E-commerce | заказы | даты, суммы | фильтрация, группировки, value_counts |
| Маркетплейс | товары + продавцы | 2 таблицы | merge, дубли ключей |
| Логистика | доставки | datetime, статусы | даты, сортировка, пропуски |
| Здоровье | пациенты + приёмы | категории | isin, строки, astype |
| Маркетинг | кампании + каналы | расходы/конверсии | pivot_table, agg, расчёт метрик |

- По 8–12 задач на датасет, от простого к сложному, с решениями
- Цель блока: гибкость — те же методы на разной предметке

**Блок 3. Рабочие задачи junior/middle**
- Retention по когортам (когорта = месяц первого заказа)
- Воронка: конверсии по шагам событийного лога
- ARPU / ARPPU / средний чек по сегментам
- Когортная таблица через pivot_table
- Каждая задача: бизнес-формулировка («продакт спрашивает...») → решение → вывод словами
- Мост к продуктовому курсу: за смыслом метрик → «Продуктовые метрики и кейсы» (кросс-селл)

**Блок 4. Мини-визуализация**
- matplotlib минимум: line (динамика), bar (сравнение), hist (распределение)
- Подписи осей, заголовок, размер — всё, без zoo графиков
- 3–4 задачи: построить график к уже решённым задачам блока 3

## Практика

Весь модуль — практика. Объём: 20 (практикум) + ~50 (датасеты) + 4 рабочие + 4 визуализация.
