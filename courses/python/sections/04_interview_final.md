## Ключевые тезисы

**Как проходит Python-секция на собесе аналитика**
- Форматы: live-coding в шаринге экрана / ноутбук с задачами / устные вопросы по коду
- Что оценивают: ход мысли вслух, а не только ответ; реакция на подсказки
- Статистика: pandas 30–40% собесов, алгоритмы ~20% (мост на модуль 5 расширенного тарифа)

**Tricky-задачи pandas (типовые ловушки middle-уровня)**
- merge с дублями ключей: почему строк стало больше
- Поведение NaN: NaN != NaN, агрегации пропускают NaN, groupby теряет NaN-ключи
- apply vs векторизация: когда apply оправдан
- Сортировка с NaN, value_counts с dropna
- Копия vs view, chained assignment (SettingWithCopyWarning)
- Формат: задача → попробуй сам → разбор ловушки

**Задачи из сборника реальных компаний**

- pandas-задачи из сборника → сюда; алгоритмические → в модуль 5 (расширенный)

**Частые ошибки и best practices**
- Читаемость: имена, разбивка цепочек, комментарии к шагам
- Чистый код в ноутбуке: ячейка = шаг анализа, вывод после каждой
- Чек-лист самопроверки кода перед показом интервьюеру

**Шпаргалка pandas ↔ SQL**
