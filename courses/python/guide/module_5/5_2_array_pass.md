# 5.2 Приём 1: проход по массиву

Один цикл + аккумулятор. Этим приёмом решается большинство алго-задач уровня «аналитик»: максимумы, суммы, частоты, уникальность.

---

## Универсальный шаблон

```python
def solve(arr):
    result = ...            # 1. аккумулятор: число / список / словарь / set
    for x in arr:           # 2. один проход
        ...                 # 3. обновление аккумулятора по условию
    return result
```

Вся вариативность — в выборе аккумулятора:

| Вопрос задачи | Аккумулятор |
|---|---|
| максимум / минимум / сумма | число |
| «сколько раз встречается» | словарь частот |
| «есть ли дубликаты», «уникальные» | set |
| «собери подходящие» | список |

---

## Базовые примеры

**Максимум без max():**

```python
def find_max(arr):
    mx = arr[0]
    for x in arr:
        if x > mx:
            mx = x
    return mx
# O(n) по времени, O(1) по памяти
```

**Частоты через словарь (ручной Counter):**

```python
def count_freq(arr):
    freq = {}
    for x in arr:
        freq[x] = freq.get(x, 0) + 1
    return freq
# O(n) время, O(n) память
```

`freq.get(x, 0)` возвращает 0, если ключа ещё нет — компактнее, чем if/else из модуля 1. Оба варианта валидны на собесе.

**Есть ли дубликаты — set как память о «виденном»:**

```python
def has_duplicates(arr):
    seen = set()
    for x in arr:
        if x in seen:        # O(1) проверка!
            return True
        seen.add(x)
    return False
# O(n) время, O(n) память
```

> 💡 **Главная мысль:** set с именем `seen` — главный персонаж алго-секции. «Видел ли я этот элемент раньше?» за O(1) — на этом строится огромный класс решений.

## Классика: второй максимум

```python
def second_max(arr):
    mx = second = None
    for x in arr:
        if mx is None or x > mx:
            second = mx          # старый максимум становится вторым
            mx = x
        elif x != mx and (second is None or x > second):
            second = x
    return second
# O(n) за ОДИН проход — сильнее, чем два прохода из модуля 1
```

**Edge cases, которые озвучь сам:** пустой список → None; все элементы одинаковые → None; дубли максимума `[20, 20, 10]` → 10.

## Классика: пара с заданной суммой (two sum)

«Есть ли в массиве два числа, дающие в сумме target?» — самая известная задача собеседований.

```python
# ❌ Лобовое решение: O(n²)
def two_sum_slow(arr, target):
    for i in range(len(arr)):
        for j in range(i + 1, len(arr)):
            if arr[i] + arr[j] == target:
                return True
    return False

# ✅ Через set: O(n)
def two_sum(arr, target):
    seen = set()
    for x in arr:
        if target - x in seen:    # нужное дополнение уже встречалось?
            return True
        seen.add(x)
    return False
```

Ход рассуждения, который стоит проговорить интервьюеру: «для каждого x мне нужно target − x. Вместо поиска по всему массиву буду запоминать пройденное в set и проверять за O(1)».

🎯 **Мини-практика:** реши одним проходом: 1) сумма и среднее без sum(); 2) индекс первого отрицательного числа; 3) самый частый элемент (частоты + максимум по значению словаря). Для каждого назови сложность вслух.
