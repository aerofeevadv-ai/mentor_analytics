# ⚡ Модуль 5. Алгоритмы для собеседований 🔒

Алгоритмическая секция без паники. Два приёма — проход по массиву и два указателя — это ядро алго-задач, которые дают аналитикам: из них собирается большинство секций.

> 📌 **Контекст:** алгоритмы спрашивают на ~20% собесов аналитиков, чаще всего в бигтехе (особенно в Яндексе). Уровень — easy и лёгкий medium LeetCode. Деревья, графы и динамическое программирование аналитикам не дают — их в модуле нет, и это осознанно: 20/80.

## Путь модуля

- [5.1 Big-O на пальцах](5_1_big_o.md) — отвечаем на вопрос «какая сложность?»
- [5.2 Приём 1: проход по массиву](5_2_array_pass.md) — один цикл решает почти всё: максимум, частоты, топ-N
- [5.3 Приём 2: два указателя](5_3_two_pointers.md) — убираем вложенный цикл
- [5.4 Как вести себя на алго-секции](5_4_interview_behavior.md) — скрипт от условия до ответа
- [5.5 Тренировочный набор: 22 задачи](5_5_training_set.md) — три уровня, к каждой задаче подсказка, решение и сложность

## Критерий завершения модуля

- решаешь задачи разогрева и основы без подсказок
- для каждого решения называешь сложность
- прошёл тренировочный набор дважды
