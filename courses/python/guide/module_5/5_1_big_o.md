# 5.1 Сложность алгоритмов: Big-O на пальцах

После твоего решения интервьюер почти всегда спросит: «какая у него сложность?». Отвечать на это — навык на 15 минут изучения.

---

## Идея

Big-O отвечает на вопрос: **во сколько раз вырастет время работы, если данных станет в N раз больше?** Без математики — просто посмотри на структуру кода.

| Сложность | По-человечески | Как выглядит в коде |
|---|---|---|
| O(1) | «сразу» | обращение по индексу/ключу: `arr[5]`, `d[key]`, `x in my_set` |
| O(n) | «один проход» | один цикл по данным |
| O(n²) | «цикл в цикле» | вложенный цикл по тем же данным |
| O(n log n) | «сортировка» | `sorted(arr)`, `arr.sort()` |

```python
# O(1) — не зависит от размера
first = arr[0]

# O(n) — один проход
total = 0
for x in arr:
    total += x

# O(n²) — для каждого элемента перебираем все остальные
for i in range(len(arr)):
    for j in range(len(arr)):
        ...

# O(n log n) — сортировка
arr_sorted = sorted(arr)
```

## Почему dict и set «мгновенные»

Поиск в списке — O(n): Python перебирает элементы, пока не найдёт. Поиск в словаре и множестве — O(1): хэш сразу указывает на место.

```python
x in my_list   # O(n) — перебор
x in my_set    # O(1) — мгновенно
d[key]         # O(1)
```

> 💡 **Главная мысль:** половина «оптимизаций» на собеседовании — это заменить поиск по списку на поиск по set/dict, превратив O(n²) в O(n). Увидел `in список` внутри цикла — почти наверняка это и есть место для улучшения.

```python
# ❌ O(n²): для каждого элемента — поиск по списку
result = []
for x in a:
    if x in b_list:        # O(n) внутри цикла
        result.append(x)

# ✅ O(n): поиск по множеству O(1)
b_set = set(b_list)
result = []
for x in a:
    if x in b_set:
        result.append(x)
```

## Как отвечать на вопрос «какая сложность?»

Шаблон ответа из трёх частей:

> «Один проход по массиву — O(n) по времени. Дополнительно я создаю словарь размером до n — O(n) по памяти. Сортировки нет, вложенных циклов нет.»

Правила быстрой оценки:

- циклы последовательно (не вложенно) → складываются: O(n) + O(n) = O(n)
- цикл в цикле → перемножаются: O(n²)
- константы отбрасываются: «два прохода» — это всё ещё O(n)
- сортировка внутри решения → минимум O(n log n)

> ⚠️ **Типичная ошибка:** назвать «O(n)» решение, внутри цикла которого прячется `x in list` или `list.remove()` — обе операции сами по себе O(n), итого O(n²). Скрытая сложность операций со списком — любимая проверка интервьюера.
