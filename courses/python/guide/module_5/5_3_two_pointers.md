# 5.3 Приём 2: два указателя

Два индекса, движущиеся по массиву по правилам, убирают вложенный цикл: O(n²) → O(n). Второй и последний приём, который нужен аналитику.

---

## Идея

Вместо «для каждого элемента перебери все остальные» — два указателя, каждый из которых проходит массив не больше одного раза:

- **с двух концов навстречу:** left в начале, right в конце, сходятся к середине
- **параллельно по двум массивам:** i по первому, j по второму

Когда срабатывает: массив **отсортирован** (или его можно отсортировать), и условие подсказывает, какой указатель двигать.

---

## Пара с суммой в отсортированном массиве

Та же two sum, но массив отсортирован — и можно без дополнительной памяти:

```python
def two_sum_sorted(arr, target):
    left, right = 0, len(arr) - 1
    while left < right:
        s = arr[left] + arr[right]
        if s == target:
            return True
        elif s < target:
            left += 1        # сумма мала — двигаем левый (числа растут)
        else:
            right -= 1       # сумма велика — двигаем правый
    return False
# O(n) время, O(1) память
```

Логика для проговаривания: «массив отсортирован, значит если сумма крайних мала — увеличить её может только движение левого указателя вправо».

## Палиндром

«Читается ли строка одинаково в обе стороны?»

```python
def is_palindrome(s):
    left, right = 0, len(s) - 1
    while left < right:
        if s[left] != s[right]:
            return False
        left += 1
        right -= 1
    return True
# O(n) время, O(1) память
```

Частый follow-up: «а если игнорировать регистр и пробелы?» → предобработка `s = s.replace(" ", "").lower()`.

## Слияние двух отсортированных списков

Кирпичик merge sort и регулярная задача сама по себе:

```python
def merge_sorted(a, b):
    result = []
    i = j = 0
    while i < len(a) and j < len(b):
        if a[i] <= b[j]:
            result.append(a[i])
            i += 1
        else:
            result.append(b[j])
            j += 1
    # один из списков закончился — дописываем хвост другого
    result.extend(a[i:])
    result.extend(b[j:])
    return result
# O(n + m) время
```

> ⚠️ **Типичная ошибка:** забыть хвосты после цикла. Проверь на `a=[1, 5], b=[2]` — без extend пятёрка потеряется.

## Разворот списка на месте

```python
def reverse_in_place(arr):
    left, right = 0, len(arr) - 1
    while left < right:
        arr[left], arr[right] = arr[right], arr[left]   # обмен без temp-переменной
        left += 1
        right -= 1
    return arr
# O(n) время, O(1) память — «на месте», без нового списка
```

## Когда какой приём

| Сигнал в условии | Приём |
|---|---|
| частоты, дубликаты, «встречалось ли» | проход + dict/set |
| отсортированный массив + поиск пары/диапазона | два указателя с концов |
| два отсортированных массива | два указателя параллельно |
| «на месте», «без дополнительной памяти» | два указателя |
| ничего не подходит | честный проход, потом думать об оптимизации |

🎯 **Мини-практика:** 1) проверь палиндром «А роза упала на лапу Азора» (с предобработкой); 2) найди в отсортированном массиве пару с суммой 0; 3) слей `[1,3,5]` и `[2,3,4,9]`, посчитай сложность.
