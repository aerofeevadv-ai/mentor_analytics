## Легенда

Ты аналитик e-commerce компании. CEO попросил подготовить обзор бизнеса к квартальной встрече. У тебя три источника: заказы + пользователи (E-commerce), доставки (Логистика), рекламные кампании (Маркетинг).

## Часть 1. Продажи (orders.csv + users.csv)

1. Загрузи и почисти данные: дубли, пропуски, отрицательные цены, грязные города. Зафиксируй комментарием каждое решение по чистке
2. Ключевые цифры: выручка, число заказов, покупателей, средний чек. Динамика по месяцам
3. Разрезы: топ-категории и топ-города по выручке, вклад сегментов
4. Retention первого месяца по когортам — улучшается или ухудшается?

## Часть 2. Доставка (deliveries.csv)

5. Почисти статусы и даты, посчитай срок доставки
6. Среднее и медианное время по складам; доля доставок дольше 7 дней
7. Найди узкое место: какой склад/курьер портит метрики сильнее всего

## Часть 3. Маркетинг (campaigns.csv)

8. Посчитай CAC и ROMI по каналам (от сумм, не среднее от строк!)
9. Какие кампании сжигают бюджет без конверсий? Сколько денег можно сэкономить?
10. Куда перераспределить бюджет — дай рекомендацию с цифрами

## Часть 4. Выводы и графики

11. Построй 3 графика: динамика выручки (line), топ-городов (bar), распределение сроков доставки (hist) — с заголовками и подписями осей
12. Напиши в финальной ячейке резюме для CEO: 5–7 предложений, только цифры и рекомендации, без кода

## Чек-лист самопроверки

**Техника:**
- [ ] Restart & Run All проходит без ошибок
- [ ] дубли и пропуски обработаны, решения прокомментированы
- [ ] размеры проверены после каждого merge
- [ ] нет циклов по строкам DataFrame и chained assignment
- [ ] метрики-отношения посчитаны от сумм

**Аналитика:**
- [ ] каждая часть заканчивается выводом словами
- [ ] цифры в резюме сходятся с расчётами выше
- [ ] рекомендации опираются на данные, а не на ощущения

**Оформление:**
- [ ] ячейка = шаг, markdown-заголовки между частями
- [ ] графики с заголовком и осями
- [ ] имена переменных читаются
