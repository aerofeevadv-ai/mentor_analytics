Устный формат: читаешь код → отвечаешь вслух → открываешь разбор. Именно так спрашивают на собесах.

## Дрил 1. Mutable default argument

```python
def add_item(item, lst=[]):
    lst.append(item)
    return lst

print(add_item(1))
print(add_item(2))
print(add_item(3))
```

❓ Что напечатает каждый из трёх вызовов?

<details>
<summary>Ответ и разбор</summary>

```
[1]
[1, 2]
[1, 2, 3]
```

Список `lst=[]` создаётся один раз — при объявлении функции, а не при каждом вызове. Все три вызова добавляют в один и тот же объект.

Правило: не использовать изменяемые объекты (list, dict, set) как дефолтные аргументы. Стандартный паттерн:

```python
def add_item(item, lst=None):
    if lst is None:
        lst = []
    lst.append(item)
    return lst
```

</details>

## Дрил 2. Присваивание списка

```python
a = [1, 2, 3]
b = a
b.append(4)
print(a)
```

❓ Что напечатает?

<details>
<summary>Ответ и разбор</summary>

```
[1, 2, 3, 4]
```

`b = a` не копирует список — обе переменные указывают на один объект в памяти. `b.append(4)` мутирует тот же объект, что видит `a`.

Чтобы сделать независимую копию:

```python
b = a.copy()      # поверхностная копия
b = a[:]          # срез — тоже поверхностная копия
import copy; b = copy.deepcopy(a)  # глубокая
```

</details>

## Дрил 3. is vs ==

```python
x = [1, 2, 3]
y = [1, 2, 3]
print(x == y)
print(x is y)
```

❓ Что напечатает каждая строка?

<details>
<summary>Ответ и разбор</summary>

```
True
False
```

`==` сравнивает значения; `is` сравнивает идентичность объектов (адрес в памяти). Два отдельно созданных списка с одинаковым содержимым — равны по `==`, но разные объекты по `is`.

Правило: `is` использовать только для сравнения с None (`if x is None`). Для значений — всегда `==`.

⚠️ Частая ловушка: для небольших целых чисел Python кеширует объекты, поэтому `a = 5; b = 5; a is b` даёт `True` — но это детали реализации CPython, на них нельзя полагаться.

</details>

## Дрил 4. Изменение списка во время итерации

```python
nums = [1, 2, 3, 4, 5]
for n in nums:
    if n % 2 == 0:
        nums.remove(n)
print(nums)
```

❓ Чего ты ожидаешь? Что получится на самом деле?

<details>
<summary>Ответ и разбор</summary>

```
[1, 3, 5]
```

Похоже на правильный ответ, но это случайность — 4 не удалена! При итерации Python двигает внутренний индекс: удалив 2 (индекс 1), следующий шаг пропускает элемент с индексом 2 (это был 3). Потом удаляет 4 (индекс 3), пропускает 5... Конкретный результат зависит от порядка.

Правильные способы:

```python
nums = [n for n in nums if n % 2 != 0]   # list comprehension
# или
nums = list(filter(lambda n: n % 2 != 0, nums))
```

</details>

## Дрил 5. Срез списка — копия или нет?

```python
original = [1, 2, 3, 4, 5]
sliced   = original[1:3]
sliced[0] = 99
print(original)
print(sliced)
```

❓ Изменится ли `original`?

<details>
<summary>Ответ и разбор</summary>

```
[1, 2, 3, 4, 5]
[99, 3]
```

Срез создаёт **новый список** (поверхностную копию). Изменение `sliced` не затрагивает `original`.

Запомни контраст с pandas: `df[mask]` — это **не** всегда копия, отсюда SettingWithCopyWarning. Срез питоновского списка — всегда копия.

</details>

## Дрил 6. np.nan == np.nan

```python
import numpy as np
print(np.nan == np.nan)
print(np.nan is np.nan)
```

❓ Что напечатает?

<details>
<summary>Ответ и разбор</summary>

```
False
True
```

`np.nan` — это один конкретный объект float (стандарт IEEE 754: NaN не равен сам себе). `is` возвращает `True` потому что обе стороны ссылаются на один объект `np.nan` в памяти — но это не стоит использовать для проверки.

Правильный способ проверить NaN: `pd.isna(x)` или `np.isnan(x)` (только для float).

Следствие для pandas: `df[df["col"] == np.nan]` всегда даёт пустой DataFrame — правильно `df[df["col"].isna()]`.

</details>

## Дрил 7. += внутри функции

```python
def modify(s):
    s += " world"

mystr = "hello"
modify(mystr)
print(mystr)
```

❓ Что напечатает?

<details>
<summary>Ответ и разбор</summary>

```
hello
```

Строки в Python **неизменяемые** (immutable). `s += " world"` внутри функции создаёт новый строковый объект и присваивает его локальной переменной `s`. Переменная `mystr` снаружи по-прежнему указывает на исходный объект.

Сравни с изменяемыми объектами:

```python
def modify_list(lst):
    lst += [4]     # это lst.__iadd__([4]) — мутирует объект на месте!

mylist = [1, 2, 3]
modify_list(mylist)
print(mylist)     # [1, 2, 3, 4] — список изменился!
```

`+=` для list — in-place операция; для str/int/tuple — создаёт новый объект.

</details>

## Дрил 8. Область видимости в функции

```python
x = 10

def change():
    x = 20
    print("inside:", x)

change()
print("outside:", x)
```

❓ Что напечатает?

<details>
<summary>Ответ и разбор</summary>

```
inside: 20
outside: 10
```

Присваивание `x = 20` внутри функции создаёт **локальную** переменную — оно не трогает глобальный `x`. Это LEGB: сначала ищется Local, потом Enclosing, Global, Built-in.

Чтобы изменить глобальную: `global x`. Но в аналитическом коде это антипаттерн — передавай значение через параметры и возвращай через `return`.

```python
x = 10

def change():
    global x
    x = 20

change()
print(x)   # 20
```

</details>
