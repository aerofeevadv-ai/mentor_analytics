# 0.2 Настройка окружения

Через 15 минут у тебя будет рабочее окружение: Python, Jupyter Notebook и первый выполненный код.

🎬 Видео: скринкаст «Jupyter за 10 минут» — встроить сюда

---

## Что мы установим и зачем

**Anaconda** — дистрибутив, который одной установкой даёт всё нужное:

- Python (нужная версия уже внутри)
- Jupyter Notebook — основной инструмент аналитиков
- Библиотеки pandas / numpy / matplotlib — предустановлены
- Графический интерфейс Anaconda Navigator — без возни с терминалом

**Почему Jupyter Notebook, а не «обычная» среда программирования:**

- код выполняется по ячейкам — видишь результат каждого шага сразу
- идеален для работы с таблицами и графиками
- 95% аналитиков и DS работают именно так

> 💡 **Jupyter vs IDE (VS Code, PyCharm):** IDE нужны разработчикам — большие проекты, Git, архитектура. Аналитику для исследования данных Jupyter удобнее. Если позже захочешь IDE — навыки переносятся один в один.

---

## Шаг 1. Скачай и установи Anaconda

1. Открой [anaconda.com/download](https://www.anaconda.com/download)
2. Скачай версию для своей системы (Windows / macOS)
3. Установи с настройками по умолчанию
4. Открой **Anaconda Navigator**

📷 _[скриншот: окно Anaconda Navigator]_

---

## Шаг 2. Запусти Jupyter Notebook

В Anaconda Navigator найди карточку **Jupyter Notebook** → нажми **Launch**.

Откроется вкладка браузера — это и есть твой рабочий инструмент.

📷 _[скриншот: Jupyter в браузере]_

---

## Шаг 3. Создай первый ноутбук

1. Нажми **New → Python 3 (ipykernel)** — появится файл `Untitled.ipynb`
2. В ячейку вставь код и нажми **Shift + Enter**:

```python
import pandas as pd

df = pd.DataFrame({"a": [1, 2, 3]})
df
```

Увидел таблицу с тремя строками — всё работает, окружение готово. ✅

> 🎯 **Практика:** переименуй ноутбук в `course_test`, добавь вторую ячейку с `print("Я готов")` и выполни её. Этот ноутбук — первый в твоей личной базе знаний.

## ⚠️ Частые ошибки установки

**1. Jupyter не запускается из Navigator**
Переустанови Anaconda или нажми «Reset Application» в настройках Navigator.

**2. `OSError` / `Permission denied` при загрузке файлов**
Не используй пути с русскими буквами («Документы»). Создай папку `C:\PandasCourse` (Windows) или `~/PandasCourse` (macOS) и работай в ней.

**3. `FileNotFoundError: No such file`**
Проверь: файл лежит в той же папке, где ноутбук; название совпадает 1-в-1 (регистр важен); файл виден в списке файлов Jupyter.

**4. `ModuleNotFoundError: No module named 'pandas'`**
Выполни в ячейке: `!pip install pandas` (в Anaconda почти всегда уже предустановлен).

**5. Jupyter открывается не в той папке (Windows)**
Anaconda Navigator → Launch Terminal → выполни:

```
cd C:\PandasCourse
jupyter notebook
```

## ★ Локальный vs облачный Jupyter

<details>
<summary>Развернуть</summary>

- **Локальный** работает на твоём компьютере — его мы и используем в курсе
- **Облачный** открывается по ссылке — так чаще всего устроено в компаниях
- Функционально они одинаковы, разница только в том, где хранятся файлы

Навык переносится без изменений: на работе откроешь облачный Jupyter и продолжишь как привык.

</details>

## ★ Что такое Git (коротко)

<details>
<summary>Развернуть</summary>

Git — система контроля версий: хранит историю изменений кода, помогает работать в команде и откатывать ошибки.

Аналитики в продуктовых компаниях встречают Git в связке с Airflow: ETL-скрипты хранятся в Git, обновления идут через pull request, команда ревьюит изменения.

**Для этого курса Git не нужен.** Просто знай, что он есть — пригодится на работе.

</details>
