# 0.3 Как учиться с LLM

LLM (ChatGPT, Claude и другие) — мощный ускоритель обучения. Но есть одно правило, которое определяет, поможет он тебе или навредит.

---

## Главное правило

> ⚠️ **На собеседовании LLM запрещён.** Задачи ты будешь решать при интервьюере: в шаринге экрана, руками, проговаривая ход мысли. Если во время курса за тебя решал чат — на собесе это станет очевидно за две минуты.

Поэтому:

> 💡 **LLM — наставник, не решатель.** Все задачи тренировочного трека решай сам. LLM подключай **после** своей попытки: объяснить, разобрать, проверить.

---

## Как использовать правильно

✅ **После своей попытки решения:**

- «Почему мой код не работает? Вот код и ошибка: ...»
- «Я решил задачу так: ... Это нормальное решение? Как сделать чище?»
- «Объясни по шагам, как работает это решение: ...»
- «Объясни разницу между merge и concat на простом примере»
- «Дай мне 3 похожие задачи на groupby, я хочу ещё потренироваться»

❌ **Так навык не сформируется:**

- «Реши задачу: ...» (до своей попытки)
- «Напиши код, который считает retention» (вместо себя, а не после себя)

---

## Готовые промпты для учёбы

**Разбор ошибки:**

```
Я учусь Python + pandas. Вот мой код и ошибка.
Не давай готовое решение — объясни, в чём причина ошибки
и в какую сторону думать.

Код: [вставь код]
Ошибка: [вставь текст ошибки]
```

**Ревью решения:**

```
Я решил учебную задачу. Проверь моё решение: правильно ли оно,
что можно упростить, какие подводные камни я не учёл.

Задача: [условие]
Моё решение: [код]
```

**Дополнительная тренировка:**

```
Дай мне 5 задач на [тему: фильтрация в pandas / словари / groupby]
уровня junior-аналитика, без решений. Я решу сам и пришлю на проверку.
```

---

## Зачем вообще LLM, если на собесе нельзя

В работе LLM разрешён и активно используется: ускоряет рутину, помогает с синтаксисом, объясняет чужой код. Уметь работать с ним в паре — рабочий навык.

Но фундамент он не заменяет: чтобы проверить код, который написал LLM, нужно самому понимать, что этот код делает. Курс даёт фундамент, LLM — ускорение поверх него.
