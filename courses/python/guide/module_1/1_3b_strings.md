# 1.3b Строки (str)

**Тип: неизменяемый (immutable)**

Строки в работе аналитика повсюду: ID, категории, города, грязные значения из выгрузок. Здесь — минимум методов, который закрывает 80% задач.

---

## Конкатенация и f-strings

```python
name = "Alex"

"Hello, " + name          # → "Hello, Alex" — склейка через +
"-" * 10                  # → "----------" — повторение
```

> ⚠️ **Типичная ошибка:** `"Возраст: " + 25` упадёт с TypeError — `+` склеивает только строки со строками. Либо приводи: `"Возраст: " + str(25)`, либо используй f-string.

**f-string — главный способ собрать строку:**

```python
city = "Москва"
conv = 4.456

f"Город: {city}"               # → "Город: Москва"
f"Конверсия: {conv:.1f}%"      # → "Конверсия: 4.5%" — :.1f округляет до 1 знака
```

Буква `f` перед кавычками, переменные в фигурных скобках. Типы приводятся сами, форматирование чисел встроено. Для выводов, отчётов и print-отладки — только f-strings.

---

## Методы, которые реально нужны

```python
s = "  Москва, Питер, Казань  "

s.strip()                    # убрать пробелы по краям → "Москва, Питер, Казань"
s.lower()                    # к нижнему регистру
s.replace("Питер", "СПб")    # замена подстроки

"a,b,c".split(",")           # строка → список → ['a', 'b', 'c']
", ".join(["a", "b", "c"])   # список → строка → "a, b, c"
```

Связка `strip()` + `lower()` — стандартная чистка грязных категорий: `" Москва "` и `"москва"` становятся одним значением.

> ⚠️ **Напоминание из 1.2:** строки неизменяемы — методы возвращают НОВУЮ строку. `s.strip()` без сохранения результата ничего не меняет: пиши `s = s.strip()`.

## Срезы и `in`

Строка — последовательность символов, индексация и срезы работают как у списков:

```python
s = "analyst"

s[0]       # → "a"
s[-1]      # → "t"
s[:3]      # → "ana"
s[::-1]    # → "tsylana" — разворот, тот же приём из 1.3
```

Проверка подстроки — через `in`:

```python
"моск" in "Москва".lower()   # → True
```

В SQL это `LIKE '%моск%'` — здесь короче и читабельнее.

📌 **Классика собеседований — палиндром:** строка читается одинаково в обе стороны. Решение — одна строка: `s == s[::-1]`. Для фраз сначала почисти: `s.lower().replace(" ", "")`.

## 🐼 Мост к pandas

> 💡 **Главная мысль:** всё из этого раздела вернётся в модуле 2 как `.str`-методы: `df["city"].str.lower()`, `.str.strip()`, `.str.replace()`, `.str.split()` — та же логика, применённая сразу ко всей колонке. Выучил методы здесь — получил чистку текстовых колонок бесплатно.

🎯 **Мини-практика:** дана строка `s = "  Data, Python, SQL  "`. Убери пробелы по краям, приведи к нижнему регистру, разбей в список по `", "`. Затем напиши проверку, является ли фраза `"А роза упала на лапу Азора"` палиндромом (не забудь про регистр и пробелы).

<details>
<summary>✅ Решение</summary>

```python
s = "  Data, Python, SQL  "
parts = s.strip().lower().split(", ")
print(parts)   # → ['data', 'python', 'sql']

phrase = "А роза упала на лапу Азора"
clean = phrase.lower().replace(" ", "")
print(clean == clean[::-1])   # → True
```

</details>
