# 1.2 Типы данных

У каждого значения в Python есть тип. Тип определяет, что с этим значением можно делать — и где код сломается, если перепутать.

---

## Базовые типы

**int / float — числа**

```python
a = 10      # int: целое
b = 3.14    # float: дробное
```

Зачем аналитику: вычисления, агрегаты, метрики. В данных `int` и `float` ведут себя по-разному: колонка с пропусками автоматически становится float — встретишь это в pandas.

**str — строки**

```python
s = "product_123"
```

Зачем аналитику: ID, категории, города, тексты. Главный источник проблем: число `"100"` в кавычках — это строка, и `"100" + 1` упадёт с TypeError.

**bool — логический тип**

```python
x = True   # или False
```

Зачем аналитику: результаты сравнений. Вся фильтрация в pandas построена на bool: `df["age"] > 30` возвращает колонку из True/False.

**None — отсутствие значения**

```python
result = None
```

Зачем аналитику: аналог NULL в SQL и NaN в pandas. «Здесь ничего нет» — это не ноль и не пустая строка, это отдельное значение.

---

## Изменяемые и неизменяемые типы

В Python два класса типов, и эта разница важнее, чем кажется.

**Неизменяемые (immutable):** int, float, str, bool, None, tuple
После создания значение изменить нельзя — любая операция создаёт новый объект.

**Изменяемые (mutable):** list, dict, set
Объект можно менять «на месте»: добавлять, удалять, перезаписывать элементы.

```python
# Строка неизменяема — операция создаёт НОВУЮ строку
s = "abc"
s.upper()
print(s)        # "abc" — оригинал не изменился
s2 = s.upper()  # результат надо сохранить

# Список изменяем — операция меняет ЕГО САМОГО
items = [1, 2, 3]
items.append(4)
print(items)    # [1, 2, 3, 4] — оригинал изменился
```

> 💡 **Главная мысль:** ключевой вопрос при любой операции — «я изменил объект или создал новый?». В pandas это вернётся: одни методы меняют DataFrame на месте, другие возвращают копию. Кто не понял разницу здесь, будет терять изменения там.

> ⚠️ **Типичная ошибка:** `s.upper()` без сохранения результата. Метод отработал, вернул новую строку, она улетела в никуда. Для неизменяемых типов результат всегда сохраняй: `s = s.upper()`.

## Сводная таблица

| Тип | Что хранит | Изменяемый | Зачем аналитику |
|---|---|---|---|
| int | целые числа | нет | счётчики, количества |
| float | дробные | нет | метрики, доли, деньги |
| str | текст | нет | ID, категории |
| bool | True/False | нет | условия, фильтры |
| None | «ничего» | нет | пропуски (NULL/NaN) |
| list | последовательность | **да** | наборы значений |
| dict | ключ → значение | **да** | маппинги |
| tuple | последовательность | нет | фиксированные наборы |
| set | уникальные элементы | **да** | дедупликация, пересечения |

Коллекции из нижней половины таблицы разберём по одной в следующих разделах.

🎯 **Мини-практика:** объясни словами (себе вслух), почему `nums.append(4)` не требует `nums = nums.append(4)`, а `s = s.upper()` требует сохранения. Если можешь объяснить — концепция твоя.
