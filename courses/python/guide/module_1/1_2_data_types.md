# 1.2 Типы данных

У каждого значения в Python есть тип. Тип определяет, что с этим значением можно делать — и где код сломается, если перепутать.

---

## Базовые типы

**int / float — числа**

```python
a = 10      # int: целое
b = 3.14    # float: дробное
```

Зачем аналитику: вычисления, агрегаты, метрики. В данных `int` и `float` ведут себя по-разному: колонка с пропусками автоматически становится float — встретишь это в pandas.

**str — строки**

```python
s = "product_123"
```

Зачем аналитику: ID, категории, города, тексты. Главный источник проблем: число `"100"` в кавычках — это строка, и `"100" + 1` упадёт с TypeError.

**bool — логический тип**

```python
x = True   # или False
```

Зачем аналитику: результаты сравнений. Вся фильтрация в pandas построена на bool: `df["age"] > 30` возвращает колонку из True/False.

**None — отсутствие значения**

```python
result = None
```

Зачем аналитику: аналог NULL в SQL и NaN в pandas. «Здесь ничего нет» — это не ноль и не пустая строка, это отдельное значение.

---

## Арифметика

```python
a = 7
b = 3

a + b    # 10   сложение
a - b    # 4    вычитание
a * b    # 21   умножение
a / b    # 2.33... обычное деление — результат ВСЕГДА float
a // b   # 2    целочисленное деление (дробная часть отбрасывается)
a % b    # 1    остаток от деления
a ** b   # 343  степень
```

Приоритет — как в математике: сначала `**`, потом `* / // %`, потом `+ -`. Сомневаешься — ставь скобки: `(a + b) * 2`.

> ⚠️ **Типичная ошибка для SQL-человека:** в Python `7 / 3` даёт `2.33`, а не `2`. В многих SQL-диалектах деление целых даёт целое, здесь — нет. Для целочисленного деления есть отдельный оператор `//`.

**`+=` — увеличить и перезаписать:**

```python
count = 0
count += 1    # то же самое, что count = count + 1
count -= 2    # уменьшить
total = 10
total *= 1.2  # умножить (например, +20% к цене)
```

`+=` будет везде: счётчики в циклах, подсчёт частот в словарях. Встретишь его уже в разделе 1.4.

## Приведение типов

Тип значения можно сменить явно:

```python
int("100")      # 100 — строка → целое
float("3.14")   # 3.14 — строка → дробное
str(42)         # "42" — число → строка
```

Классическая ситуация — число приехало строкой (из CSV, из формы, из API):

```python
"100" + 1        # ❌ TypeError: can only concatenate str (not "int") to str
int("100") + 1   # ✅ 101

"100" + "1"      # "1001" — строки склеиваются, это не сложение!
```

> 📌 **Запомни:** `+` для строк — склейка, для чисел — сложение. Если «сложение» дало `"1001"` вместо `101` — у тебя строки, проверь `type()`. В pandas та же операция называется `astype()` — колонку-строку приводят к числу перед расчётами.

## Изменяемые и неизменяемые типы

В Python два класса типов, и эта разница важнее, чем кажется.

**Неизменяемые (immutable):** int, float, str, bool, None, tuple
После создания значение изменить нельзя — любая операция создаёт новый объект.

**Изменяемые (mutable):** list, dict, set
Объект можно менять «на месте»: добавлять, удалять, перезаписывать элементы.

```python
# Строка неизменяема — операция создаёт НОВУЮ строку
s = "abc"
s.upper()
print(s)        # "abc" — оригинал не изменился
s2 = s.upper()  # результат надо сохранить

# Список изменяем — операция меняет ЕГО САМОГО
items = [1, 2, 3]
items.append(4)
print(items)    # [1, 2, 3, 4] — оригинал изменился
```

> 💡 **Главная мысль:** ключевой вопрос при любой операции — «я изменил объект или создал новый?». В pandas это вернётся: одни методы меняют DataFrame на месте, другие возвращают копию. Кто не понял разницу здесь, будет терять изменения там.

> ⚠️ **Типичная ошибка:** `s.upper()` без сохранения результата. Метод отработал, вернул новую строку, она улетела в никуда. Для неизменяемых типов результат всегда сохраняй: `s = s.upper()`.

## Алиасинг: `b = a` — это не копия

Следствие изменяемости, на котором валятся на собеседованиях. Что выведет этот код?

```python
a = [1, 2, 3]
b = a
b.append(4)
print(a)   # ?
```

<details>
<summary>Ответ</summary>

`[1, 2, 3, 4]`. Изменили `b`, а поменялся и `a`.

`b = a` НЕ копирует список — обе переменные указывают на ОДИН объект в памяти. Это две таблички с именами на одной коробке: что положил через `b`, лежит и в `a`.

</details>

Нужна независимая копия — проси её явно:

```python
a = [1, 2, 3]
b = a.copy()   # настоящая копия
b.append(4)
print(a)       # [1, 2, 3] — оригинал цел
print(b)       # [1, 2, 3, 4]
```

С неизменяемыми типами (числа, строки) такой проблемы нет: их нельзя поменять «на месте», поэтому `b = a` безопасно.

> 📌 **Любимый tricky-вопрос интервьюеров:** «что выведет код?» с `b = a` для списка или словаря. Теперь ты знаешь ответ. В pandas то же самое вернётся как разница между view и copy.

## Сводная таблица

| Тип | Что хранит | Изменяемый | Зачем аналитику |
|---|---|---|---|
| int | целые числа | нет | счётчики, количества |
| float | дробные | нет | метрики, доли, деньги |
| str | текст | нет | ID, категории |
| bool | True/False | нет | условия, фильтры |
| None | «ничего» | нет | пропуски (NULL/NaN) |
| list | последовательность | **да** | наборы значений |
| dict | ключ → значение | **да** | маппинги |
| tuple | последовательность | нет | фиксированные наборы |
| set | уникальные элементы | **да** | дедупликация, пересечения |

Коллекции из нижней половины таблицы разберём по одной в следующих разделах.
