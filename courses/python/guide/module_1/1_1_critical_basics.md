# 1.1 Критические основы

Девять правил, без которых не работает ничего остального. 90% ошибок новичков — отсюда.

---

## 1. Отступы — это синтаксис, а не оформление

Python использует отступы вместо скобок. Отступ говорит интерпретатору, какой код находится «внутри» условия, цикла или функции.

```python
# ❌ Ошибка
if x > 10:
print("hi")

# ✅ Правильно
if x > 10:
    print("hi")
```

> ⚠️ **Типичная ошибка:** IndentationError — самая частая ошибка новичков. Стандарт: 4 пробела (Jupyter ставит их сам при нажатии Tab).

---

## 2. `=` против `==` — присвоить или сравнить

```python
x = 10        # = присваивание: записать значение в переменную

if x == 10:   # == сравнение: проверить, равны ли значения
    print("x равен 10")
```

```python
# ❌ Ошибка — = в условии
if x = 10:
    print("ok")
# SyntaxError: invalid syntax. Maybe you meant '==' or ':=' instead of '='?
```

Python сам подсказывает в тексте ошибки: «может, ты имел в виду `==`?». Читай ошибки — они умнее, чем кажутся.

> 💡 **Запоминалка:** один знак — кладёшь значение, два знака — задаёшь вопрос.

---

## 3. print() — главный инструмент отладки

Не понимаешь, что происходит в коде — выведи на экран:

```python
x = 10
print("x:", x)
```

В Jupyter есть бонус: последняя строка ячейки выводится сама, без `print()`.

---

## 4. Ошибки нормальны — учись их читать

Текст ошибки говорит, что именно сломалось. Читай **последнюю строку** сообщения:

| Ошибка | Причина |
|---|---|
| `NameError` | переменной не существует (опечатка или не создана) |
| `KeyError` | нет такого ключа в словаре |
| `IndexError` | вышел за пределы списка |
| `TypeError` | операция не поддерживается для этого типа |
| `SyntaxError` | опечатка в синтаксисе (скобки, двоеточие, `=` в условии) |
| `IndentationError` | проблема с отступами |

> 💡 **Главная мысль:** ошибка — не «всё сломалось», а подсказка, где смотреть. Аналитик, умеющий читать ошибки, чинит код за минуту, не умеющий — гуглит полчаса.

---

## 5. Комментарии — твой лучший друг

```python
# Это комментарий
x = 10   # комментарий можно писать справа
```

Комментируй шаги анализа: через месяц откроешь ноутбук и скажешь себе спасибо.

---

## 6. Имена переменных должны читаться

```python
# ❌ Плохо
a = 10
b = 20

# ✅ Хорошо
product_price = 10
user_age = 20
```

Стандарт Python — **snake_case**: маленькие буквы, слова через подчёркивание.

---

## 7. Python чувствителен к регистру

```python
price = 10
Price = 20   # это ДРУГАЯ переменная
```

---

## 8. Код читается сверху вниз

```python
print(a)   # ❌ NameError: переменная ещё не создана
a = 10
```

В Jupyter это коварнее: ячейки можно выполнять в любом порядке, и переменная из «нижней» ячейки может жить в памяти. Если ноутбук ведёт себя странно — `Kernel → Restart & Run All`.

---

## 9. Проверяй тип через type()

```python
print(type(10))        # <class 'int'>
print(type("hi"))      # <class 'str'>
print(type([1, 2, 3])) # <class 'list'>
```

Половина «мистических» багов в данных — это число, которое на самом деле строка. Привычка проверять тип сэкономит часы.
