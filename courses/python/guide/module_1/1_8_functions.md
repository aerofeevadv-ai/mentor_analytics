# 1.8 Функции (def)

Функция — именованный блок кода, который делает одну логическую операцию: принимает входные данные, возвращает результат.

> 📌 **Контекст:** на собеседовании почти всегда ждут именно функцию — `solve(nums)`, `two_sum(arr)`, `is_prime(n)`. Интервьюер передаёт в неё входные данные и проверяет вывод. Без уверенного владения функциями на алгоритмической секции делать нечего.

---

## Основной шаблон

```python
def имя_функции(параметры):
    # действия
    return результат
```

```python
def double(x):
    return x * 2

print(double(10))   # → 20
```

---

## return — обязателен

Если `return` нет, функция возвращает None:

```python
def wrong_double(x):
    x * 2              # вычислили, но не вернули

print(wrong_double(10))   # → None
```

> ⚠️ **Типичная ошибка:** функция «что-то печатает» через print, но не возвращает значение. На собеседовании это провал теста: проверяющий код ждёт return. Правило: функция ВОЗВРАЩАЕТ результат, print — только для отладки.

---

## Аргументы и значения по умолчанию

```python
def discount(price, rate=0.1):
    return price * (1 - rate)

discount(100)             # → 90.0  (rate по умолчанию)
discount(100, rate=0.2)   # → 80.0
```

- `price` — обязательный аргумент
- `rate=0.1` — аргумент со значением по умолчанию

---

## Возврат нескольких значений

```python
def min_and_max(nums):
    return min(nums), max(nums)

result = min_and_max([3, 7, 1, 10])
print(result)   # → (1, 10) — это кортеж

low, high = min_and_max([3, 7, 1, 10])   # сразу в две переменные
```

## Имена функций

Правила PEP8: нижний регистр, слова через подчёркивание, имя передаёт смысл.

| Паттерн | Когда использовать |
|---|---|
| `get_*` | вернуть значение |
| `calculate_*` | вычислить метрику |
| `is_*` | вернуть True/False |
| `normalize_*` | привести к стандартному виду |
| `process_*` | обработать набор данных |
| `solve()` | стандарт для алгоритмических задач |

## Практический пример: валидация данных

Типичная аналитическая функция — привести значение к норме:

```python
def normalize_price(price):
    # Убираем отрицательные значения
    if price < 0:
        price = 0

    # Округляем до копеек
    price = round(price, 2)

    # Минимальная цена
    if price < 10:
        price = 10

    return price

print(normalize_price(-5))      # → 10
print(normalize_price(12.345))  # → 12.35
print(normalize_price(5))       # → 10
```

**🐼 Мост к pandas:** именно такие функции ты будешь применять к целым колонкам через `apply()`. Написал логику для одного значения → применил к миллиону строк.

## Признаки хорошей функции

- делает одну понятную вещь
- название отражает смысл
- все пути выполнения заканчиваются return
- нет магических чисел — только понятные переменные
- легко протестировать на нескольких входах

> 💡 **Главная мысль:** функция — это контракт: «дай мне вход, я верну результат». Чистый контракт легко тестировать, переиспользовать и объяснять интервьюеру. Привычка заворачивать логику в функции отличает кандидата уровня junior+ от того, кто пишет код простынёй.
