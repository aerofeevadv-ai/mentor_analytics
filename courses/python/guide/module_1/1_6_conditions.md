# 1.6 Условия (if / elif / else)

Условия выполняют разные действия в зависимости от значений. Фундамент фильтрации данных и аналитической логики «если X → делаем Y».

---

## Базовый шаблон

```python
age = 25

if age < 18:
    result = "Несовершеннолетний"
elif age < 40:
    result = "Взрослый"
else:
    result = "Старшая возрастная группа"

print(result)   # → Взрослый
```

Как читается:

- `if` — первая проверка
- `elif` — следующая проверка, если предыдущая не сработала (может быть несколько)
- `else` — всё остальное
- выполняется **первая** подошедшая ветка, остальные пропускаются

Это тот же `CASE WHEN ... THEN ... ELSE ... END` из SQL — один в один по смыслу.

---

## Порядок проверок имеет значение

```python
score = 78

if score > 90:
    grade = "A"
elif score > 75:
    grade = "B"     # сработает: 78 > 75
else:
    grade = "C"

print(grade)   # → B
```

> ⚠️ **Типичная ошибка:** ветки проверяются сверху вниз, и срабатывает первая подошедшая. Если поставить `score > 75` ПЕРЕД `score > 90`, оценку «A» не получит никто: 95 перехватится веткой «B». В SQL CASE WHEN — та же ловушка.

## Сравнения и логические связки

```python
x == 10    # равно
x != 10    # не равно
x >= 18    # больше или равно

# Связки
if age >= 18 and city == "Москва":   # обе проверки истинны
if age < 18 or age > 65:             # хотя бы одна истинна
if not is_active:                    # отрицание
```

---

## Вложенные условия

```python
country = "RU"
age = 20

if country == "RU":
    if age >= 18:
        allowed = True
    else:
        allowed = False
else:
    allowed = None   # неизвестный случай

print(allowed)   # → True
```

Работает, но глубже двух уровней — сигнал переписать: обычно вложенность заменяется связкой `and` или ранним выходом из функции.

## 🐼 Мост к pandas

> 💡 **Главная мысль:** в pandas ты редко напишешь `if` для строк таблицы. Вместо этого — булева маска: `df["age"] >= 18` проверяет условие сразу для ВСЕХ строк и возвращает колонку True/False. Логика условий та же, исполнение векторное. `and`/`or`/`not` превратятся в `&`/`|`/`~` — запомни это место, к нему вернёмся.

🎯 **Мини-практика:** напиши код, который по сумме заказа `order_sum` присваивает сегмент: больше 10000 — «VIP», больше 3000 — «средний», иначе — «массовый». Проверь на значениях 15000, 5000, 500. Затем поменяй порядок веток и посмотри, что сломается.
