# 1.5 Кортежи (tuple) и множества (set)

Две коллекции с одной суперспособностью каждая: кортеж нельзя изменить, множество не хранит дубликаты.

---

## Кортеж (tuple)

**Тип: неизменяемый (immutable)**

Как список, но после создания менять нельзя: ни добавить, ни удалить, ни перезаписать.

```python
coords = (55.7, 37.6, 120)

coords[0]      # → 55.7 — индексация как у списка
coords[-1]     # → 120
len(coords)    # → 3

coords[1] = 10
# → TypeError: 'tuple' object does not support item assignment
```

**Когда нужен:**

- зафиксировать данные, которые не должны меняться (координаты, параметры)
- функция возвращает несколько значений — это кортеж: `return min(nums), max(nums)`
- неизменяемость делает tuple безопасным: его можно использовать как ключ словаря

**🐼 Мост к pandas:** `df.shape` возвращает кортеж `(строки, колонки)` — теперь ты знаешь, почему его нельзя изменить.

---

## Множество (set)

**Тип: изменяемый, но элементы внутри должны быть неизменяемыми**

Коллекция **уникальных** элементов. Порядок не гарантируется, индексов нет.

```python
# Дубликаты удаляются сами
unique_ids = {10, 20, 30, 30}
print(unique_ids)        # → {10, 20, 30}

# Дедупликация списка — главный приём
vals = [1, 2, 2, 3, 4, 4, 5]
unique_vals = set(vals)  # → {1, 2, 3, 4, 5}

# Добавить / удалить
unique_ids.add(40)
unique_ids.remove(20)

# Проверка принадлежности — очень быстрая
10 in unique_ids         # → True
```

**Операции теории множеств:**

```python
a = {1, 2, 3}
b = {3, 4, 5}

a & b   # пересечение → {3}
a | b   # объединение → {1, 2, 3, 4, 5}
a - b   # разность → {1, 2}
```

**Зачем аналитику:** «какие пользователи есть в обеих группах?», «кто купил в январе, но не в феврале?» — это пересечения и разности множеств. Плюс мгновенная проверка `x in allowed_ids` на больших наборах.

> ⚠️ **Типичная ошибка:** у set нет индексов — `s[0]` упадёт с TypeError. И в set нельзя положить список (изменяемый объект) — только числа, строки, кортежи.

**🐼 Мост к pandas:** концепция уникальности вернётся как `drop_duplicates()` и `nunique()`. А `isin()` в pandas — это проверка принадлежности множеству, как `IN` в SQL.

🎯 **Мини-практика:** есть два списка покупателей за январь и февраль (с повторами внутри). Найди: уникальных покупателей каждого месяца, тех, кто покупал в оба месяца, тех, кто был в январе и пропал в феврале.
