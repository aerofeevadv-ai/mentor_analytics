# 1.7b List и dict comprehensions

Comprehension — способ записать цикл «создал пустой → перебрал → добавил» одной строкой. На собеседованиях вопрос «перепишите этот цикл короче» звучит почти гарантированно — ждут именно comprehension.

---

## List comprehension

Тот же паттерн из 1.7, два варианта записи:

```python
nums = [1, 2, 3, 4, 5]

# Обычный цикл
squares = []
for n in nums:
    squares.append(n**2)

# Comprehension — то же самое одной строкой
squares = [n**2 for n in nums]   # → [1, 4, 9, 16, 25]
```

Читается слева направо: «возьми `n**2` для каждого `n` из `nums`».

**С условием — фильтрация:**

```python
nums = [3, 12, 5, 18, 1, 20]

big = [n for n in nums if n > 10]   # → [12, 18, 20]
```

> 💡 **Мост к SQL:** `[n for n in nums if n > 10]` — это `SELECT n FROM nums WHERE n > 10` одной строкой. Выражение в начале — твой SELECT (можно преобразовывать: `n * 2`), условие в конце — WHERE.

**Преобразование + фильтр вместе:**

```python
[n * 2 for n in nums if n % 2 == 0]   # удвоить только чётные → [24, 36, 40]
```

## Dict comprehension

Та же идея для словарей:

```python
d = {"a": 5, "b": 15, "c": 2, "d": 20}

# Оставить пары со значением больше 10
{k: v for k, v in d.items() if v > 10}   # → {'b': 15, 'd': 20}

# Построить словарь из списка
words = ["data", "analyst", "sql"]
{w: len(w) for w in words}               # → {'data': 4, 'analyst': 7, 'sql': 3}
```

---

## Когда comprehension НЕ нужен

Comprehension хорош, пока читается с листа. Сигналы вернуться к обычному циклу:

- внутри несколько действий, а не одно выражение
- сложная ветвящаяся логика (if/elif/else)
- хочется вложить comprehension в comprehension

> ⚠️ **Типичная ошибка:** запихнуть всю логику в одну нечитаемую строку, чтобы «выглядело профессионально». Профессионально — это когда коллега понимает код с первого взгляда. Сомневаешься — пиши цикл.

## 🐼 Мост к pandas

> 💡 **Главная мысль:** для обработки колонок comprehension не нужен — там векторные операции. Но один приём из pandas-практики встречается постоянно: отобрать колонки по условию — `[c for c in df.columns if "date" in c]`. Плюс comprehension — стандарт для подготовки списков и маппингов вокруг DataFrame.

🎯 **Мини-практика:** перепиши одной строкой задачи из 1.7: «квадраты чисел `[1, 2, 3, 4, 5]`» и «числа больше 10 из `[3, 12, 5, 18, 1, 20]`». Затем из списка `["Москва", "Питер", "Казань"]` собери словарь «город → длина названия».

<details>
<summary>✅ Решение</summary>

```python
squares = [n**2 for n in [1, 2, 3, 4, 5]]
big = [n for n in [3, 12, 5, 18, 1, 20] if n > 10]

cities = ["Москва", "Питер", "Казань"]
lengths = {c: len(c) for c in cities}
print(lengths)   # → {'Москва': 6, 'Питер': 5, 'Казань': 6}
```

</details>
