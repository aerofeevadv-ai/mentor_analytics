# 1.8b Сортировка: sorted(key=...) и lambda

Сортировка «не по самому элементу, а по его части» — выручка города, значение в словаре, второй элемент кортежа. Это закрывает целый класс задач на собеседованиях, включая топ-N частот из раздела 1.4.

---

## sorted() — основа

```python
nums = [14, 2, 9]

sorted(nums)                 # → [2, 9, 14] — новый список, оригинал цел
sorted(nums, reverse=True)   # → [14, 9, 2] — по убыванию
```

`sorted()` возвращает НОВЫЙ список (есть ещё метод `nums.sort()` — он сортирует на месте; для начала достаточно одного `sorted()`).

---

## lambda — функция без имени

```python
double = lambda x: x * 2     # то же, что def double(x): return x * 2
double(10)                   # → 20
```

`lambda` — функция из одного выражения, без `def`, без имени, `return` подразумевается. Сама по себе не нужна: её сила в передаче как аргумента — в `key=` для сортировки и в `apply()` в pandas.

> 📌 **Правило:** логика помещается в одно выражение — lambda. Сложнее — обычный `def` с понятным именем.

---

## key= — сортировать по части элемента

`key` принимает функцию, которая говорит, ПО ЧЕМУ сортировать:

```python
sales = [("Москва", 120), ("Казань", 340), ("Питер", 90)]

# По выручке (второй элемент кортежа)
sorted(sales, key=lambda x: x[1])
# → [('Питер', 90), ('Москва', 120), ('Казань', 340)]

# По выручке по убыванию
sorted(sales, key=lambda x: x[1], reverse=True)
# → [('Казань', 340), ('Москва', 120), ('Питер', 90)]
```

`lambda x: x[1]` читается: «для каждого элемента x возьми x[1] и сортируй по нему».

---

## Классика собеседований: топ-N частот

Продолжение паттерна из 1.4 — посчитали частоты, теперь достаём самые частые:

```python
freq = {"a": 3, "b": 2, "c": 1}

top = sorted(freq.items(), key=lambda x: x[1], reverse=True)
top[:2]   # → [('a', 3), ('b', 2)] — топ-2
```

Разбор: `freq.items()` даёт пары `(ключ, частота)`, сортируем по частоте по убыванию, срезом берём первые N.

> 💡 **Мост к SQL:** это `ORDER BY cnt DESC LIMIT 2`. Связка «частоты через словарь + sorted(key=) + срез» полностью закрывает задачу «найди самые частые элементы» — одну из самых популярных на собеседованиях аналитиков.

---

## 🐼 Мост к pandas

> 💡 **Главная мысль:** lambda — твой постоянный спутник в pandas: `df["price"].apply(lambda x: x * 1.2)` применяет функцию к каждому значению колонки. А сортировки в pandas встроены — `sort_values()`, и параметр направления там тот же по духу: `ascending=False` вместо `reverse=True`.

🎯 **Мини-практика:** дан список `people = [("Bob", 25), ("Ann", 31), ("Tom", 19)]`. Отсортируй по возрасту по убыванию. Затем из словаря `freq = {"x": 7, "y": 12, "z": 3}` получи топ-2 пар по значению.

<details>
<summary>✅ Решение</summary>

```python
people = [("Bob", 25), ("Ann", 31), ("Tom", 19)]
print(sorted(people, key=lambda x: x[1], reverse=True))
# → [('Ann', 31), ('Bob', 25), ('Tom', 19)]

freq = {"x": 7, "y": 12, "z": 3}
top = sorted(freq.items(), key=lambda x: x[1], reverse=True)[:2]
print(top)   # → [('y', 12), ('x', 7)]
```

</details>
