## to_csv() — запись CSV

**Что делает:** сохраняет DataFrame в файл.

```python
df.to_csv("result.csv", index=False)
```

> ⚠️ **Подводный камень:** без `index=False` pandas запишет индекс отдельной колонкой. При следующей загрузке появится мусорная колонка «Unnamed: 0». Видишь её в данных — значит, кто-то сохранял без index=False. Правило: to_csv почти всегда с index=False.

**Чуть сложнее:** выгрузка для Excel-пользователей с разделителем «;» и русской кодировкой:

```python
df.to_csv("for_excel.csv", index=False, sep=";", encoding="cp1251")
```

## read_excel() / to_excel() ★

```python
df = pd.read_excel("table.xlsx")
df = pd.read_excel("table.xlsx", sheet_name="Лист2")   # конкретный лист

df.to_excel("report.xlsx", index=False)
```

**Подводный камень:** нужен пакет openpyxl (`!pip install openpyxl`). И помни: Excel портит данные при ручном открытии CSV — длинные ID превращает в научную нотацию, даты переворачивает. Если коллега жалуется на «кривой файл» — часто дело в этом, а не в твоей выгрузке.

## 🧠 Как этим пользуется профи

Промежуточные результаты анализа сохраняет в CSV с index=False и говорящим именем (`orders_clean_2026_06.csv`), финальные отчёты для бизнеса — в Excel. Сырые данные не перезаписывает никогда: исходник священен, все трансформации воспроизводятся кодом из ноутбука.

🎯 **Практика:** 1) сохрани очищенный тренировочный датасет в CSV без индекса; 2) загрузи обратно и проверь shape — совпадает с сохранённым? 3) сохрани топ-10 по зарплате в Excel.
