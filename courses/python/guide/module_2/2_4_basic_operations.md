# 2.4 Базовые операции

Колонки, фильтрация, индекс, пропуски и главное — векторное мышление. После этой страницы DataFrame перестанет быть чёрным ящиком.

---

## Работа с колонками

```python
df["age"]                                  # одна колонка → Series
df[["name", "age"]]                        # несколько колонок → DataFrame (двойные скобки!)

df["age_plus_1"] = df["age"] + 1           # создать колонку
df = df.drop(columns=["age"])              # удалить
df = df.rename(columns={"name": "user_name"})   # переименовать
```

> ⚠️ **Типичная ошибка:** `df["name", "age"]` без внутренних скобок → KeyError. Несколько колонок — это СПИСОК имён внутри скобок: `df[["name", "age"]]`.

В SQL это просто `SELECT name, age` и `AS` — здесь то же самое, другой синтаксис.

---

## Базовая фильтрация: булева маска

Сердце pandas. Условие на колонку возвращает Series из True/False:

```python
df["age"] > 30
# 0     True
# 1    False
# 2     True
# ...
```

Подставь маску в `df[...]` — останутся строки, где True:

```python
df[df["age"] > 30]              # WHERE age > 30
df[df["city"] == "Moscow"]      # WHERE city = 'Moscow'
```

> 💡 **Главная мысль:** фильтрация в pandas — это всегда два шага: построить маску из True/False, применить её к таблице. Все сложные фильтры (модуль B2) — комбинации этой пары.

## Индекс

- индекс — это **метка строки**, по умолчанию 0, 1, 2...
- метка ≠ позиция: после фильтрации индексы «дырявые» (0, 3, 7...), и это нормально

```python
df.index               # посмотреть индекс
df = df.reset_index(drop=True)   # сбросить на 0,1,2... (drop=True — не сохранять старый колонкой)
df = df.set_index("id")          # сделать колонку индексом
```

## Пропуски и типы

Пропуски — это `NaN`/`None`, аналог NULL в SQL:

```python
df.isna()          # маска пропусков по всей таблице
df.isna().sum()    # количество пропусков по колонкам — главная команда диагностики
```

Пропуски ломают вычисления и фильтры, поэтому их ищут в первую очередь (обработка — в [B6](2_m06_missing.md)).

Типы колонок:

| dtype | Что это |
|---|---|
| int64 / float64 | числа |
| object | строки (и всё подозрительное) |
| bool | логические |
| datetime64 | даты |

> ⚠️ **Типичная ошибка:** колонка с числами имеет тип object — значит, где-то в ней затесалась строка (пробел, «н/д», число с запятой вместо точки). Агрегации по ней будут врать или падать.

## Векторные операции — главное отличие от чистого Python

```python
df["price_with_tax"] = df["price"] * 1.2
```

Это **не цикл**. Операция применилась ко всей колонке сразу:

- код короче и читается как формула
- работает в сотни раз быстрее цикла
- pandas «думает массивами», а не отдельными значениями

```python
# ❌ Так не надо (стиль новичка, медленно)
for i in range(len(df)):
    df.loc[i, "price_with_tax"] = df.loc[i, "price"] * 1.2

# ✅ Так надо
df["price_with_tax"] = df["price"] * 1.2
```

> 💡 **Главная мысль:** увидел в своём коде цикл по строкам DataFrame — остановись и поищи векторный способ. Он есть почти всегда.
