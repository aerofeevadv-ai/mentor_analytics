# B7. Дубликаты

Дубли — вторая после пропусков причина «поехавших» метрик: задвоенные заказы завышают выручку, задвоенные пользователи ломают конверсию.

**SQL-аналог группы:** `DISTINCT`, `GROUP BY ... HAVING COUNT(*) > 1`.

---

## duplicated() — поиск дублей

**Что делает:** возвращает маску True для строк-повторов.

```python
df.duplicated().sum()                  # сколько полных дублей строк
df[df.duplicated()]                    # посмотреть на них глазами
df.duplicated(subset=["user_id"]).sum()   # дубли по конкретной колонке
```

**Чуть сложнее:** по умолчанию первое вхождение НЕ считается дублем (keep="first"). Посмотреть все копии, включая первую:

```python
df[df.duplicated(subset=["user_id"], keep=False)]
```

**Подводный камень:** «полный дубль строки» и «дубль по ключу» — разные проверки. Заказы с одинаковым user_id — норма (человек купил дважды), две строки с одинаковым order_id — проблема. Думай, ЧТО должно быть уникальным в этой таблице.

---

## drop_duplicates() — удаление дублей

**Что делает:** удаляет повторяющиеся строки.

```python
df = df.drop_duplicates()                          # полные дубли строк
df = df.drop_duplicates(subset=["user_id"])        # по ключу, остаётся первая
df = df.drop_duplicates(subset=["user_id"], keep="last")   # остаётся последняя
```

**SQL:** точно соответствует `SELECT DISTINCT` только для полных дублей строк. Для `subset+keep` — честный аналог `ROW_NUMBER() OVER (PARTITION BY user_id ORDER BY ...) = 1`: удаляем «лишние» строки, оставляя одну по условию.

**Чуть сложнее:** «оставить последнюю запись по пользователю» — типовая задача (актуальный статус, дата последней регистрации):

```python
df = df.sort_values("signup_date").drop_duplicates(subset=["user_id"], keep="last")
```

> ⚠️ **Подводный камень:** drop_duplicates без сортировки оставляет ПЕРВУЮ ПОПАВШУЮСЯ строку — какая раньше лежала в файле, та и выжила. Если важно, какая запись останется — сначала sort_values, потом drop_duplicates. Частая ошибка в задачах «возьми последний статус».

---

## 🧠 Как этим пользуется профи

Перед любым merge и любым подсчётом уникальных: `df.duplicated(subset=[ключ]).sum()`. Ноль — работаем дальше. Не ноль — смотрим на дубли глазами (keep=False) и выясняем причину, а не молча сносим.

🎯 **Практика:** в `data.csv`: 1) проверь количество полных дублей строк и дублей по `user_id` — они разные, объясни почему; 2) посмотри на все копии задублированных пользователей (keep=False); 3) оставь по каждому `user_id` запись с максимальной `signup_date`; 4) объясни, чем отличался бы результат без предварительной сортировки.
