# B9. Работа с текстом

Строковые колонки — это категории, ID, города, названия. Грязный текст = неправильные группировки: «Moscow», «moscow» и «Moscow » станут тремя разными городами.

**SQL-аналог группы:** `LIKE`, `LOWER`/`UPPER`, `TRIM`.

Все строковые методы вызываются через приставку `.str`:

---

## str.contains() — поиск подстроки

**Что делает:** маска True для строк, содержащих подстроку.

```python
df[df["product"].str.contains("iPhone")]
```

**SQL:** `WHERE product LIKE '%iPhone%'`

**Чуть сложнее:**

```python
df[df["product"].str.contains("iphone", case=False)]        # без учёта регистра
df[df["product"].str.contains("iPhone", na=False)]          # NaN считать как False
```

> ⚠️ **Подводный камень:** если в колонке есть NaN, str.contains вернёт NaN вместо False, и фильтр упадёт с ошибкой про NA. Лекарство: параметр `na=False`. Сэкономит тебе полчаса как минимум один раз.

---

## str.lower() / str.upper() — регистр

```python
df["city"] = df["city"].str.lower()
```

**SQL:** `LOWER(city)`

**Когда используется:** первая операция при унификации категорий. «Moscow» и «moscow» должны стать одним значением до группировки.

---

## str.strip() — обрезка пробелов

```python
df["city"] = df["city"].str.strip()
```

**SQL:** `TRIM(city)`

**Когда используется:** невидимый пробел в конце — классика ручного ввода и выгрузок из Excel. Глазами не видно, группировка сломана.

---

## Стандартный конвейер чистки текста

```python
df["city"] = df["city"].str.strip().str.lower()
```

> 💡 **Главная мысль:** перед ЛЮБОЙ группировкой по строковой колонке — strip + lower + value_counts глазами. Три секунды, которые спасают отчёт.

---
