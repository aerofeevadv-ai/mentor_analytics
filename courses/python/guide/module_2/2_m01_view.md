# B1. Просмотр данных

Методы первичного анализа DataFrame. Они уже встречались в части A — здесь собраны карточками с рабочими кейсами и подводными камнями, чтобы возвращаться как к справочнику.

**SQL-аналог группы:** ближе всего `LIMIT`, но честно — в SQL такого инструментария нет, это преимущество pandas.

---

## head() — первые строки

**Что делает:** показывает первые N строк (по умолчанию 5).

```python
df.head()
df.head(10)   # больше строк
```

**Когда используется:** проверка корректности загрузки, контроль результата после merge / groupby / фильтрации, быстрый взгляд на данные.

---

## tail() — последние строки

```python
df.tail()
df.tail(3)
```

**Когда используется:** временные данные (последние даты), проверка, что файл не обрезан, контроль сортировки.

---

## sample() — случайные строки

```python
df.sample(5)
```

**Чуть сложнее:** на больших таблицах sample используют для исследования без прогона всего датасета:

```python
df.sample(100000)   # сэмпл для отладки логики
```

**Подводный камень:** head() показывает «причёсанное» начало файла, проблемы живут в середине. sample() — честный взгляд в случайное место.

---

## info() — структура таблицы

**Что делает:** количество строк и колонок, тип каждой колонки, число непустых значений, объём памяти.

```python
df.info()
```

**Когда используется:** главный метод диагностики. Выявляет ошибки загрузки (числа → object), показывает пропуски по всем колонкам разом.

---

## describe() — базовая статистика

**Что делает:** среднее, std, min/max, квартили по числовым колонкам.

```python
df.describe()
```

**Чуть сложнее:** describe по строковым колонкам — счётчики и топ-значение:

```python
df.describe(include="object")
```

**Подводный камень:** describe молча пропускает NaN. Если в колонке 90% пропусков, статистика посчитается по оставшимся 10% — и будет выглядеть прилично.

**Когда используется:** поиск аномалий и выбросов (возраст 250, цена −1), проверка распределений.

---

## dtypes — типы данных

```python
df.dtypes
```

**Подводный камень:** object — самый проблемный тип. Числовая колонка с типом object означает грязь внутри: пробелы, «н/д», запятые в числах. Ошибки в dtypes → ошибки в фильтрах, агрегациях, merge.

---

## shape — размер таблицы

```python
df.shape   # → (строк, колонок), это кортеж
```

**Когда используется:** контроль размера после фильтрации и merge. «Было 10 000 строк, после merge стало 45 000» — сигнал дублей в ключе (разберём в B5).

---

## 🧠 Как этим пользуется профи

shape → head() → info() → describe() → sample(). 20–30 секунд, и таблица «прочитана». Этот порядок — твой открывающий ход на любом live-coding.

🎯 **Практика:** 3 задачи на тренировочном датасете — определи без подсказок: сколько строк и колонок; в какой колонке больше всего пропусков; есть ли подозрительные значения в числовых колонках.
