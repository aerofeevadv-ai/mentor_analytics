# B2. Фильтрация данных

Фундамент pandas: отбор пользователей, событий, товаров — почти любая задача начинается с фильтра.

**SQL-аналог группы:** `WHERE`, `IN`, `BETWEEN`, `NOT`.

---

## Булевы выражения — основа

**Что делает:** проверяет каждую строку, возвращает Series из True/False.

```python
df["age"] > 30          # маска
df[df["age"] > 30]      # применили маску → отфильтрованная таблица
```

**SQL:** `WHERE age > 30`

**Когда используется:** везде. Отбор по возрасту, доходу, типу события, дате.

---

## Составные условия: & | ~

**Что делают:** объединяют условия. `&` — И, `|` — ИЛИ, `~` — НЕ.

```python
df[(df["age"] > 30) & (df["city"] == "Moscow")]
df[(df["age"] > 25) | (df["salary"] > 80000)]
df[~df["city"].isna()]
```

**SQL:** `AND`, `OR`, `NOT`

> ⚠️ **Подводный камень №1 всего pandas:** каждое условие — в скобках. Без скобок `df[df["age"] > 30 & df["city"] == "Moscow"]` падает с непонятной ошибкой про ambiguous truth value. Привычка: написал `&` или `|` — проверь скобки вокруг обоих условий.

Заметь: не `and`/`or`/`not` из модуля 1, а именно символы `&`/`|`/`~` — потому что операция применяется к целым колонкам.

---

## isin() — принадлежность списку

**Что делает:** проверяет, входит ли значение в список допустимых.

```python
df[df["city"].isin(["Moscow", "SPB"])]
```

**SQL:** `WHERE city IN ('Moscow', 'SPB')`

**Чуть сложнее:** список обычно приходит извне — задан заранее или получен из другой таблицы:

```python
allowed_cities = ["Moscow", "SPB", "Kazan"]
df[df["city"].isin(allowed_cities)]

# Обратный вариант: NOT IN
df[~df["city"].isin(["Moscow", "SPB"])]
```

**Когда используется:** фильтр по списку ID, выбор нескольких категорий, отбор по типам устройств.

---

## between() — диапазон ★

**Что делает:** короткая запись диапазона, границы включаются.

```python
df[df["age"].between(20, 30)]
# эквивалент: df[(df["age"] >= 20) & (df["age"] <= 30)]
```

**SQL:** `WHERE age BETWEEN 20 AND 30`

Опционально: писать короче приятно, но обязательным не является.

---

## query() — SQL-подобный синтаксис

**Что делает:** фильтрация строкой-выражением, без скобочной арифметики.

```python
df.query("age > 30 and city == 'Moscow'")
df.query("city in ['Moscow', 'SPB'] and salary >= 80000")
```

**SQL:** читается почти как WHERE — поэтому тебе зайдёт.

**Подводные камни:**

- строки внутри — в другие кавычки, чем сама строка query (`"... city == 'Moscow'"`)
- имена колонок с пробелами не работают
- в query пишут `and`/`or`, а не `&`/`|` — да, ровно наоборот относительно масок. Запомни: маски — символы, query — слова

---

## 🧠 Как этим пользуется профи

Простой фильтр → булева маска. Два-три условия → маска со скобками. Список значений → isin(). Громоздкое условие, которое сложно читать → query().

🎯 **Практика:** на тренировочном датасете: 1) пользователи 25–40 лет из Москвы или Питера; 2) все, КРОМЕ городов из списка `["Moscow", "SPB"]`, без пропусков в зарплате; 3) перепиши фильтр №1 через query(). Сверь количество строк во всех вариантах.
