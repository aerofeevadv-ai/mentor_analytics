# 2.1 DataFrame и Series

Pandas — библиотека для работы с табличными данными. Если совсем просто: **Excel на стероидах**, которым управляешь кодом.

---

## Две структуры, на которых держится всё

**DataFrame** — двумерная таблица:

- строки и колонки, у каждой колонки своё имя и тип
- данные выровнены по индексу

```
index | name | age
0     | Anna | 23
1     | Bob  | 34
```

**Series** — одна колонка как отдельная структура:

- одномерная, имеет индекс
- именно её ты получаешь, когда берёшь `df["age"]`

| Характеристика | DataFrame | Series |
|---|---|---|
| Размерность | 2D | 1D |
| Что это | таблица | колонка |
| Когда используется | работа с таблицей | работа с одной колонкой |

> 💡 **Главная мысль:** DataFrame — это набор Series с общим индексом. Series — это список с индексом и методами. Ты уже знаешь списки и словари, так что здесь нет ничего нового — только новая обёртка.

Для тебя как SQL-аналитика: DataFrame — это результат твоего SELECT, живущий в памяти ноутбука.

---

## Создание DataFrame

Три главных способа:

```python
import pandas as pd

# 1) Из словаря списков (ключ = имя колонки)
df = pd.DataFrame({
    "name": ["Anna", "Bob"],
    "age": [23, 34]
})

# 2) Из списка словарей (словарь = строка)
df = pd.DataFrame([
    {"name": "Anna", "age": 23},
    {"name": "Bob", "age": 34}
])

# 3) Из списка списков + имена колонок
df = pd.DataFrame(
    [[1, "A"], [2, "B"]],
    columns=["id", "group"]
)
```

В работе ты чаще будешь загружать данные из файлов и БД, но создание DataFrame руками нужно постоянно: проверить гипотезу, собрать маленький справочник, воспроизвести багованный пример.

> ⚠️ **Типичная ошибка:** в словаре списков длины списков должны совпадать. `{"a": [1, 2], "b": [1]}` упадёт с ValueError.

🎯 **Мини-практика:** создай DataFrame с тремя сотрудниками (name, department, salary) всеми тремя способами. Убедись, что результат одинаковый.
