# 2.3 Первичный анализ данных

Центральный навык части A: **читать таблицу глазами** до того, как писать код. Опытный аналитик оценивает незнакомый датасет за 20–30 секунд.

---

## Минимальный набор

```python
df.head()        # первые строки (по умолчанию 5)
df.tail()        # последние строки
df.sample(5)     # случайные строки
df.info()        # структура: типы, пропуски, память
df.describe()    # статистика числовых колонок
df.dtypes        # типы данных колонок
df.shape         # (строк, колонок)
```

Через эти методы ты понимаешь:

- сколько данных
- какие типы у колонок (и не съехали ли они: числа строками?)
- есть ли пропуски
- есть ли аномальные значения
- как вообще устроена таблица

---

## Порядок профи

> 💡 **Главная мысль:** у первичного анализа есть устоявшийся порядок. Доведи его до рефлекса — это первое, что ты сделаешь на live-coding собеседовании, и интервьюер это заметит.

1. **`df.shape`** — размер: с чем имеем дело, тысяча строк или десять миллионов
2. **`df.head()`** — глазами: что за колонки, как выглядят значения
3. **`df.info()`** — типы и пропуски: где object вместо чисел, где дыры
4. **`df.describe()`** — распределения: минимумы, максимумы, выбросы (цена −1? возраст 250?)
5. **`df.sample(5)`** — контрольный взгляд в случайное место таблицы

> ⚠️ **Типичная ошибка:** начинать считать метрики, не посмотрев info(). Полчаса дебага сводятся к тому, что «выручка» оказалась строкой с пробелами, а половина дат — пропуски.

Подробные карточки каждого метода — в [B1. Просмотр данных](2_m01_view.md).
