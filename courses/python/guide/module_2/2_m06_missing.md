# B6. Пропуски в данных

NaN — постоянный спутник реальных данных. Уметь находить, диагностировать и осознанно обрабатывать пропуски — базовая гигиена аналитика.

**SQL-аналог группы:** `IS NULL`, `IS NOT NULL`, `COALESCE`.

---

## isna() / notna() — поиск пропусков

**Что делает:** возвращает маску True там, где пропуск (isna) или значение (notna).

```python
df.isna()                    # маска по всей таблице
df.isna().sum()              # пропуски по колонкам — главная диагностика
df[df["salary"].isna()]      # строки С пропуском зарплаты
df[df["salary"].notna()]     # строки БЕЗ пропуска
```

**SQL:** `WHERE salary IS NULL` / `IS NOT NULL`

> ⚠️ **Подводный камень:** `df["salary"] == None` — это не ошибка Python, но и не работает: сравнение молча вернёт False по всем строкам, не подняв исключения. Получишь пустую фильтрацию без единого предупреждения — что коварнее прямой ошибки. NaN не равен ничему, даже самому себе (`np.nan == np.nan` → False). Только `isna()`/`notna()`. Классический tricky-вопрос собеседований.

---

## fillna() — заполнение пропусков

**Что делает:** заменяет пропуски заданным значением.

```python
df["salary"] = df["salary"].fillna(0)
df["city"] = df["city"].fillna("unknown")
df["salary"] = df["salary"].fillna(df["salary"].median())   # медианой
```

**SQL:** `COALESCE(salary, 0)`

> ⚠️ **Подводный камень — смысловой, не технический:** заполнять надо осознанно. Залил зарплату нулями → среднее обвалилось, выводы поехали. Пропуск «не указал» и пропуск «равно нулю» — разные вещи. Прежде чем заполнять, ответь: почему здесь пропуск и что значение должно означать.

---

## dropna() — удаление пропусков

**Что делает:** удаляет строки (или колонки) с пропусками.

```python
df = df.dropna()                        # строки, где есть ХОТЬ ОДИН пропуск
df = df.dropna(subset=["salary"])       # только где пропущена salary
```

**SQL:** `WHERE salary IS NOT NULL`

**Подводный камень:** dropna() без subset — оружие массового поражения: одна дырявая второстепенная колонка, и ты выбросил половину датасета. Почти всегда нужен subset.

---

## Стратегия работы с пропусками

1. **Найди:** `df.isna().sum()` — сразу после загрузки
2. **Пойми причину:** ошибка выгрузки? необязательное поле? left merge без пары?
3. **Реши осознанно:**

| Ситуация | Решение |
|---|---|
| Пропусков <1–2% и они случайны | dropna(subset=[...]) |
| Категория неизвестна | fillna("unknown") — отдельной категорией |
| Число, нужна устойчивость | fillna медианой |
| Пропуск несёт смысл (нет покупок = 0 покупок) | fillna(0) — только если уверен |
| Пропусков много (>20–30%) | не латать, а разбираться с источником |

> 💡 **Главная мысль:** на собеседовании за «я бы удалил пропуски» без вопроса «почему они возникли» снижают оценку. Правильный ответ всегда начинается с диагностики, а не с действия.

🎯 **Практика:** в `data.csv`: 1) найди колонку с максимумом пропусков и объясни, почему там может быть пропуск (по смыслу данных); 2) заполни `city` значением "unknown", `salary` — медианой; 3) удали строки, где пропущен `email` — объясни, почему именно там можно удалять (email = необязательное поле, без него строка всё ещё несёт data.csv бизнес-смысл; `user_id` в этом датасете не имеет пропусков); 4) проверь, что в `salary` и `city` пропусков не осталось.
