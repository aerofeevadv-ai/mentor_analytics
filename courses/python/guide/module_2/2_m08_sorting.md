# B8. Сортировка

Короткая группа из одного метода, но с парой ловушек.

**SQL-аналог группы:** `ORDER BY`.

---

## sort_values() — сортировка по значениям

**Что делает:** сортирует строки по колонке (или нескольким).

```python
df.sort_values("salary")                          # по возрастанию
df.sort_values("salary", ascending=False)         # по убыванию
```

**SQL:** `ORDER BY salary DESC`

**Чуть сложнее:** по нескольким колонкам с разным направлением:

```python
df.sort_values(["city", "salary"], ascending=[True, False])
# города по алфавиту, внутри города — зарплаты по убыванию
```

**SQL:** `ORDER BY city ASC, salary DESC`

**Топ-N — типовая связка:**

```python
df.sort_values("salary", ascending=False).head(10)   # топ-10 зарплат
```

**Подводные камни:**

- sort_values возвращает копию: без `df = df.sort_values(...)` порядок не изменился
- **NaN при сортировке всегда улетают в конец** — и при ascending=True, и при False. «Топ-10 по убыванию» не покажет пропуски, а «топ-10 по возрастанию» — тоже нет. Управление: `na_position="first"`
- после сортировки индекс перемешан (5, 2, 9...) — для дальнейшей работы по позициям сделай reset_index(drop=True)

## ★ sort_index() — сортировка по индексу

<details>
<summary>Развернуть</summary>

```python
df.sort_index()
```

Пригождается после groupby/pivot_table, когда результат отсортирован не так, как хочется (например, месяцы в алфавитном порядке вместо календарного).

</details>
