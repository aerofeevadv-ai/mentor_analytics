## Как пользоваться этой картой

Три шага:
1. Открой чек-лист «Где ты сейчас» (стр. 2) -- определи свой уровень.

Карта основана на реальных собеседованиях и офферах. Зарплаты -- медианы по Москве (hh.ru + данные учеников), март 2026. В регионах и на удалёнке -- минус 15-30%.

## Карта развития аналитика

### Уровень 1. Стажёр / Junior (0-1 год)

**Зарплата:** 60 000 -- 120 000 р/мес

| Область | Что нужно уметь |
|---|---|
| SQL | SELECT, JOIN, GROUP BY, подзапросы, CASE WHEN. Решать задачи за 20 минут |
| Excel / Google Sheets | Сводные таблицы, ВПР/INDEX-MATCH, базовые графики, условное форматирование |
| Python | Основы pandas: чтение файлов, фильтрация, группировка, простые графики (matplotlib) |
| Метрики | DAU/MAU, конверсия, retention -- знать определения и уметь посчитать |
| Визуализация | Один BI-инструмент на базовом уровне: DataLens, Tableau Public или Power BI |
| Soft skills | Умение задавать уточняющие вопросы, структурировать ответ на собесе |

**Как попасть на этот уровень:**
- Пройти курс по SQL (Stepik, Karpov.Courses, LeetCode -- 50+ задач)
- Сделать 2-3 pet-проекта с открытыми данными (Kaggle)
- Составить резюме с двумя версиями: Data Analyst и Product Analyst
- Начать откликаться -- первые 50 откликов покажут, где пробелы

**Типичный путь:** 3-6 месяцев самостоятельной подготовки или 2-4 месяца с ментором.

---

### Уровень 2. Middle (1-3 года)

**Зарплата:** 150 000 -- 250 000 р/мес

| Область | Что нужно уметь |
|---|---|
| SQL | Оконные функции, CTE, оптимизация запросов. ClickHouse или PostgreSQL |
| Python | pandas уверенно, автоматизация отчётов, когортный анализ в коде, визуализация (seaborn/plotly) |
| Продуктовые метрики | Retention, воронки, unit-экономика (CAC, LTV, Payback Period), деревья метрик |
| A/B тесты | Дизайн эксперимента, расчёт размера выборки, p-value, типичные ошибки (peeking, SRM) |
| BI | Проектирование дашбордов: не просто "построить", а выбрать метрики, структуру, drill-down |
| Кейсы | Решение продуктовых кейсов: метрика упала -- что делаешь? Фреймворк КМДГП |
| Soft skills | Работа со стейкхолдерами, самостоятельная постановка задач, презентация результатов |

**Что отличает middle от junior:**
- Junior выполняет задачи. Middle формулирует задачи сам.
- Junior строит дашборд по ТЗ. Middle решает, какой дашборд нужен.
- Junior считает метрику. Middle объясняет, почему она изменилась и что с этим делать.

**Как вырасти с junior до middle:**
- Научиться A/B тестам -- это спрашивают на 90% собесов в продуктовую аналитику
- Разобраться в продуктовых метриках глубже: не "знаю что такое retention", а "могу декомпозировать падение retention по когортам"
- Прорешать 10-15 продуктовых кейсов
- Получить опыт работы со стейкхолдерами (или имитировать в pet-проекте)

**Типичный путь:** 1-2 года на позиции junior + целенаправленная подготовка к собесам.

### Уровень 3. Senior (3-5 лет)

**Зарплата:** 250 000 -- 400 000 р/мес

| Область | Что нужно уметь |
|---|---|
| SQL + инфраструктура | Сложные запросы, ETL-пайплайны, Airflow, понимание DWH-архитектуры |
| Python + статистика | Бутстрап, байесовские методы, линеаризация ratio-метрик, causal inference (DiD, PSM) |
| Продуктовое мышление | Проектирование системы метрик с нуля, приоритизация инициатив через data, стратегические рекомендации |
| A/B тесты (продвинуто) | Множественное тестирование, sequential testing, ratio-метрики, network effects |
| Менеджмент | Менторство джуниоров, ревью кода и дашбордов, построение процессов в команде |
| Коммуникация | Презентация для C-level, аргументация решений данными, работа с неопределённостью |

**Что отличает senior:**
- Влияет на продуктовые решения, а не просто предоставляет данные.
- Проектирует аналитические процессы, а не просто в них участвует.
- Его рекомендации меняют приоритеты команды.

**Как вырасти с middle до senior:**
- Углубить статистику: бутстрап, байесовские подходы, causal inference
- Взять ответственность за метрики продукта или направления
- Начать менторить -- это ускоряет собственный рост
- Освоить инфраструктуру данных: ETL, Airflow, DWH

**Типичный путь:** 2-3 года на middle-позиции + лидерство в проектах.

---

### Уровень 4. Lead / Head of Analytics (5+ лет)

**Зарплата:** 350 000 -- 600 000+ р/мес

| Область | Что нужно уметь |
|---|---|
| Стратегия | Выбор метрик для бизнеса, построение data-культуры в компании |
| Управление | Найм, онбординг, развитие аналитиков. Управление командой 3-10 человек |
| Архитектура | Проектирование аналитической инфраструктуры: какие инструменты, какие процессы |
| Бизнес | Понимание P&L, unit-экономики бизнеса, влияние аналитики на revenue |
| Коммуникация | Регулярные отчёты CEO/CPO, защита бюджета, кросс-функциональное влияние |

Этот уровень -- про управление и стратегию, не про хардскиллы. Код пишешь всё меньше, влияешь всё больше.

## Чек-лист: определи свой уровень

Поставь галочку напротив каждого навыка, который ты можешь уверенно продемонстрировать на собеседовании (не "слышал", а "сделаю за 20 минут").

### SQL

- [ ] SELECT, WHERE, JOIN, GROUP BY -- пишу свободно
- [ ] Подзапросы и CASE WHEN -- использую регулярно
- [ ] Оконные функции (ROW_NUMBER, LAG/LEAD, SUM OVER) -- знаю, когда применять
- [ ] CTE, оптимизация запросов -- могу объяснить план выполнения
- [ ] ClickHouse / специфика колоночных БД

### Python

- [ ] Основы: переменные, циклы, функции
- [ ] pandas: чтение данных, фильтрация, группировка, merge
- [ ] Визуализация: matplotlib или seaborn
- [ ] Когортный анализ и автоматизация отчётов в pandas
- [ ] Статистические тесты: t-test, хи-квадрат, bootstrap

### Продуктовые метрики

- [ ] DAU/WAU/MAU -- знаю разницу, могу посчитать
- [ ] Retention -- считаю по когортам, знаю про плато
- [ ] Воронки -- нахожу узкое место, сегментирую
- [ ] ARPU, ARPPU, LTV, CAC -- знаю формулы и связи
- [ ] Дерево метрик -- декомпозирую Revenue или DAU на 2-3 уровня
- [ ] Unit-экономика -- CAC, LTV/CAC, Payback Period

### A/B тесты

- [ ] Зачем нужны, базовая механика (контроль/тест)
- [ ] Расчёт размера выборки, MDE, статистическая значимость
- [ ] Типичные ошибки: peeking, SRM, novelty effect
- [ ] Продвинутое: множественное тестирование, sequential, ratio-метрики

### Продуктовые кейсы

- [ ] "Метрика упала на 15% -- что делаешь?" -- могу структурно разобрать
- [ ] Фреймворк решения кейсов (контекст, метрика, декомпозиция, гипотезы, проверка)
- [ ] Могу решить кейс по незнакомому продукту за 15 минут

### Soft skills и опыт

- [ ] Работаю со стейкхолдерами -- формулирую задачу, а не жду ТЗ
- [ ] Презентую результаты нетехнической аудитории
- [ ] Менторю джуниоров / провожу code review

### Как читать результат

**0-8 галочек** -- ты на уровне Стажёр/Junior. Фокус: SQL + базовые метрики + первые pet-проекты.

**9-16 галочек** -- ты на уровне Junior, двигаешься к Middle. Фокус: A/B тесты + продуктовые кейсы + Python.

**17-23 галочки** -- ты Middle. Фокус: продвинутая статистика + менторство + лидерство в проектах.

**24-27 галочек** -- ты Senior или близко. Фокус: управление, стратегия, рост в Lead.
