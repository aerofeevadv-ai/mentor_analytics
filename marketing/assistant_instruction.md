# Инструкция для помощницы: новая схема работы

> Дата: март 2026
> От: Алексей

---

## Что меняется

Мы переходим на новую схему работы с контентом. Вот главные изменения:

1. **Посты готовятся на 2 недели вперёд** (раньше — по ходу)
2. **Ты берёшь на себя коммуникацию с подрядчиками** (монтажёр, дизайнер)
3. **Появляются новые площадки** (IG, VK, TikTok, MAX)

---

## 1. Контент: двухнедельные спринты

### Как это работает

```
Claude готовит черновики постов на 2 недели (6-8 шт)
        ↓
Алексей получает Google Doc, читает, ставит ✅ или правки
        ↓
Ты забираешь апрувнутые посты → раскидываешь в SMMPlanner
        ↓
SMMPlanner публикует по расписанию
```

### Google Doc

Каждые 2 недели — новый документ в папке Google Drive:

```
Контент публикация/
├── Текущий спринт/
│   └── 2026-04-01_посты.gdoc
└── Архив/
    ├── 2026-03-15_посты.gdoc
    └── ...
```

В документе по каждому посту:
- Текст поста
- Тип (полезный / нарративный / анонс)
- Площадки (TG+MAX / IG / VK / все)
- Дата публикации
- Статус: ✅ апрув / правки

### Расписание публикаций

| День | TG + MAX | IG | VK |
|------|----------|----|----|
| Вт | Пост (кейс или образовательный) | Тот же пост (картинка-заглушка + caption) | Тот же пост (текст) |
| Чт | Пост (анонс видео или short) | — | — |
| Вс | Пост (личный инсайт / лёгкий) | — | — |

**TG + MAX:** 3 раза в неделю (Вт / Чт / Вс)
**IG экспертный пост:** 1 раз в неделю (вторник, = вторничный TG-пост)
**VK экспертный пост:** 1 раз в неделю (вторник, = вторничный TG-пост)

### MAX

MAX — зеркало Telegram. Все TG-посты дублируются 1:1. Настроить в SMMPlanner — публикация одновременно TG + MAX.

## 2. Shorts / Reels / Клипы

Когда монтажёр присылает готовые shorts (2-3 штуки с каждого лонга):

| Площадка | Что делать | Частота |
|----------|-----------|---------|
| YouTube Shorts | Монтажёр заливает сам (или ты) | По готовности |
| IG Reels | Залить тот же файл | 2-3/нед |
| TikTok | Залить тот же файл | 3-4/нед |
| VK Клипы | Залить тот же файл | 2-3/нед |

Один файл → 4 площадки. Можно через SMMPlanner или вручную.

## 5. Коммуникация с подрядчиками

### Главное правило

**Алексей общается только с тобой. Ты общаешься с подрядчиками.**

Если нужно решение от Алексея — пишешь ему. Всё остальное — сама.

### Монтажёр

**Что делает:** монтаж лонгов + нарезка 2-3 shorts с каждого лонга.
**SLA:** 5 рабочих дней после получения видео.
**Он надёжный** — просто нужно:
- Отправить ему видео после записи Алексеем
- Проверить черновик монтажа (чек-лист ниже)
- Если всё ок — дать добро на финализацию
- Если нужны правки — написать конкретно что исправить
- Получить финальный файл + shorts → залить на площадки

**Чек-лист проверки черновика монтажа:**
- [ ] Начало без длинного интро (сразу хук)
- [ ] Нет длинных пауз / «э-э-э»
- [ ] Вставки/графика на месте
- [ ] Субтитры читаемые
- [ ] Звук ровный (нет скачков громкости)
- [ ] Длительность в пределах плана

### Дизайнер

**Что делает:** обложки для YouTube (2 штуки в месяц).
**SLA:** 2 рабочих дня до публикации.
**Он надёжный** — просто нужно:
- Отправить ТЗ на обложку (Claude готовит, лежит в папке видео)
- Проверить результат — текст читаемый, лицо видно, не слишком много элементов
- Если ок — забрать файл
- Если нужны правки — написать конкретно

### Если подрядчик нарушает SLA

Если монтажёр или дизайнер не укладываются в срок:
1. Напомнить один раз
2. Если через сутки нет ответа — написать Алексею

## 7. Что НЕ нужно делать

- Не пишешь тексты постов (это делает Claude → Алексей апрувит)
- Не придумываешь темы
- Не общаешься с подписчиками / не отвечаешь на комментарии
- Не принимаешь решения по контенту — только публикуешь апрувнутое
- Не мониторишь аналитику площадок (пока)
